Featured Project

The OMEGA Project

A distributed, mixed-bearer network for hydrographic sensing, edge validation, and remote data ingestion. OMEGA bridges the gap between field hardware and interactive web mapping.

Explore Architecture GitHub Repository

System Architecture

OMEGA spans multiple layers of technology, from the foundational Arduino sensing logic in the water to the React-based map portal in the browser.

Layer 1: Sensing

Sensing & Validation

The foundational logic that powers OMEGA. Arduino-based acoustic logging, GPS tracking, and IMU stabilization. Includes sound-speed correction based on water temperature, and rigid edge-validation to ensure only high-quality data reaches the mapping phase.

Overview Sensing Validation IDW Mapping
Layer 2: Transport

Mesh Network

A mixed-bearer communication backbone. The Mesh Interface Registry maps physical LoRa nodes to logical IDs. A Python-based CBOR bridge handles binary deserialization, while JSONL endpoints handle TCP ingestion from remote gateways.

Mesh Architecture
Layer 3: Fleet

Hardware Edge

The physical components of the OMEGA network. Featuring the ASV Mobile Gateway (Jetson-powered), the Shore Controller for centralized command, and floating LilyGO T-Beam relays that dynamically extend the network's range over water.

Fleet Overview
Layer 4: Access

Portal & Data

The user-facing OMEGA-wave web portal and its SQLite data backend. The portal features Mapbox GL JS for interactive bathymetry overlays and wayfinding. The data ingestion engine normalizes live telemetry and pulls from external APIs like NOAA and Global Fishing Watch.
