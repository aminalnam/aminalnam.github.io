Current focus

OMEGA: Oceanic Measurement & Environmental Geospatial Array

A compact underwater survey system that records depth, position, and environmental data from a moving ROV and turns those measurements into bathymetric maps and georeferenced outputs.

Capture GPS, sonar depth, temperature, IMU orientation, SD logging, field filtering
Processing Python scripts for mapping, interpolation, contour generation, and Earth overlays
Output Survey-quality datasets, depth maps, contour maps, KML files, and overlays
Featured project

OMEGA brings sensing, filtering, and mapping into one working system.

The project combines an Arduino-based capture device with a Python-based processing workflow. During acquisition, measurements are filtered using GPS quality, fix age, platform orientation, speed, and spacing constraints; afterward, the resulting mapping dataset is converted into depth maps, interpolated surfaces, contour maps, and georeferenced outputs.

Featured system

Oceanic Measurement & Environmental Geospatial Array

OMEGA is structured to separate capture, validation, and reconstruction into distinct stages. That architecture makes the project useful both as a field tool and as a clear technical case study.

The public repository includes the Arduino logger, Python mapping scripts, documentation, example data, and a visual pipeline diagram for the full workflow.

Capture Arduino logger with GPS, sonar, temperature, IMU, LCD, and SD
Processing CSV parsing, filtering, IDW interpolation, contour generation
Outputs Depth maps, continuous surfaces, KML points, and Earth overlays

System at a glance

  • Quick start commands Depth maps, interpolated maps, contour maps, KML output, and Google Earth overlays are all exposed directly in the repo workflow.
  • Repository structure Code, docs, scripts, and example data are separated cleanly so the project is readable as both software and documentation.
  • Method Filtering is handled during acquisition, then surface reconstruction is handled during processing.
  • Usefulness The project is easy to enter at multiple levels: concept, hardware, code, outputs, or data.
