Educational Deep Dive: Hardware Strategy

COTS vs Custom Silicon

A critical decision in systems engineering is determining when to design custom printed circuit boards (PCBs) versus when to leverage Commercial Off-The-Shelf (COTS) hardware. For the OMEGA edge nodes, we explicitly chose to build upon existing COTS platforms rather than manufacturing custom silicon.

The Engineering Tradeoff

Time to Deployment

Designing custom RF hardware for the 915 MHz band requires rigorous impedance matching, Vector Network Analyzer (VNA) tuning, and FCC certification testing. By utilizing pre-certified COTS boards like the LILYGO T-Beam Supreme, we bypass months of hardware iteration and immediately focus on the core value: the firmware and mesh protocol.

Platform Selection

The T-Beam Supreme

The T-Beam Supreme was selected because it perfectly integrates our required triad into a single tested package: an ESP32-S3 microcontroller, a Semtech SX1262 LoRa transceiver, and a u-blox MAX-M10S GNSS module, all backed by an AXP2101 power management unit.

Educational Deep Dive: Sensor Wiring

I2C Bus Capacitance

The I2C protocol was designed for communication between chips on the same circuit board. In marine engineering, we often abuse this protocol by running I2C cables 5 meters down into the ocean. If you use standard 10kΩ pull-up resistors, your sensors will silently fail.

The Core Concept

Parasitic Capacitance ($C_p$)

As a copper wire gets longer, it acts like a capacitor. When the microcontroller tries to release the I2C data line to let it float HIGH (3.3V), the wire's capacitance slowly "charges up" instead of instantly snapping to 3.3V. This rounds off the square wave, causing read errors.

The Physics

The RC Time Constant

The time ($T$) it takes for the voltage to rise is determined by the equation: $T = R \times C_p$. Since we cannot easily lower the capacitance ($C_p$) of a physical 5-meter underwater cable, our only option is to drastically lower the pull-up resistance ($R$).

Step-by-Step Implementation: Calculating Pull-Up Resistors

Problem: You want to run a BME280 sensor on a 5-meter tether using 100kHz standard mode I2C. The maximum allowed rise time ($t_r$) for 100kHz I2C is $1000\text{ns}$.

  1. Estimate Cable Capacitance: A typical shielded twisted pair cable has a capacitance of ~50pF per meter. $$C_p = 5\text{m} \times 50\text{pF/m} = 250\text{pF}$$
  2. Calculate Maximum Resistance ($R_{max}$): Using the standard I2C formula $R_{max} = \frac{t_r}{0.8473 \times C_p}$: $$R_{max} = \frac{1000 \times 10^{-9}}{0.8473 \times 250 \times 10^{-12}} \approx 4721 \Omega$$
  3. Calculate Minimum Resistance ($R_{min}$): The resistor must not allow more than 3mA of sink current ($I_{OL}$) when the chip pulls the line low. Assuming a 3.3V ($V_{cc}$) system with a max $0.4V$ low-level voltage ($V_{OL}$): $$R_{min} = \frac{V_{cc} - V_{OL}}{I_{OL}} = \frac{3.3V - 0.4V}{3\text{mA}} = 966 \Omega$$
  4. Select the Resistor: Your pull-up resistor must be between $966\Omega$ and $4.7\text{k}\Omega$. To ensure crisp square waves against underwater noise, select a strong pull-up near the minimum, such as $2.2\text{k}\Omega$. Do NOT use the default $10\text{k}\Omega$ built into most breakout boards!
Embedded C++

T-Beam Supreme Architecture.

Located in firmware/tbeam_supreme_dual_sensor/, the C++ codebase is highly optimized for the ESP32-S3 microcontroller. The firmware aggressively manages deep-sleep power states, shutting down the GNSS (GPS) module and the Semtech SX1262 LoRa radio between transmission windows.

Core

ESP32-S3

Dual-core XTensa LX7 @ 240MHz. Handles TLV compression and mesh logic.

RF Transceiver

Semtech SX1262

LoRa CSS modulation. Capable of +22dBm output for over-the-horizon links.

Location

u-blox MAX-M10S

Ultra-low-power GNSS. Acquires hot-fixes in <2 seconds before sleeping.

Thermodynamics: A remote sensor floating in the Pacific cannot plug into a wall outlet. The ESP32-S3 draws roughly 100mA while active, draining a standard 18650 lithium battery in two days. By forcing the hardware into the ULP (Ultra Low Power) sleep state, the draw drops to 15µA.

Edge UI & Telemetry

OLED Display & PMU Diagnostics

Deploying a headless edge node into the ocean is dangerous if field technicians cannot visually confirm its operational state before launch. The latest firmware introduces a robust local UI using an onboard SH1106 OLED screen and direct I2C communication with the AXP2101 Power Management Unit (PMU).

  • Dynamic Field Observability: The OLED dynamically renders the assigned operator_label, mesh-id, and physical MAC address directly on the screen.
  • Live Hardware Diagnostics: By querying the internal ESP32-S3 die temperature and the AXP2101 PMU, the screen reports real-time battery voltage, charging status, and thermal metrics.
  • GPS Rendering: Latitude and longitude coordinates are painted to the display the moment the u-blox GNSS module acquires a hot fix.
void update_oled_display() {
    display.clear();
    // 1. Render Identity
    display.drawString(0, 0, "OP: " + String(operator_label));
    display.drawString(0, 10, "ID: " + String(mesh_id));
    
    // 2. Render Hardware Health
    float batt_v = pmu.getBatteryVoltage();
    float core_temp = temperatureRead();
    display.drawString(0, 20, "BAT: " + String(batt_v) + "V  TMP: " + String(core_temp) + "C");
    
    // 3. Render Position
    display.drawString(0, 30, "GPS: " + String(gps.location.lat(), 4) + ", " + String(gps.location.lng(), 4));
    
    display.display();
}
