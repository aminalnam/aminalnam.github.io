Educational Deep Dive: Hardware Strategy

COTS vs Custom Silicon

A critical decision in systems engineering is determining when to design custom printed circuit boards (PCBs) versus when to leverage Commercial Off-The-Shelf (COTS) hardware. For the OMEGA edge nodes, we explicitly chose to build upon existing COTS platforms rather than manufacturing custom silicon.

Sensor Node

T-Beam Supreme

The core data collection node. Integrates an ESP32-S3, an AXP2101 PMU, and a Semtech SX1262. During integration, we bypassed manufacturer specification errors to map the correct SX1261 pinmap (NSS 41, DIO1 40) directly.

Long Range Node

T-Beam 1W

The high-power variant featuring an external HPA for +30 dBm EIRP. Firmware-level cooldown algorithms run the cooling fan after transmissions to dissipate residual heat. Its upside-down OLED requires custom rotation logic.

Mesh Relay

T3-S3 LR1121

A sensor-less mesh router capable of communicating simultaneously across Sub-GHz and 2.4 GHz LoRa bands, crucial for establishing high-bandwidth bulk-transfer handshakes across disparate networks.

Shore Gateway

T-ETH-ELITE

The DTO v3.0 physical edge gateway. A LilyGo T-ETH-ELITE baseboard stacked with a T-SX1302 8-channel concentrator. It catches bursts of telemetry across 8 simultaneous channels and forwards them via Semtech UDP over Ethernet.

RF Physics & Networking

Cross-Family Mesh Interoperability.

Getting the Semtech SX126x (Supreme/1W) and LR11x0 (T3-S3) chip families to communicate over a LoRa mesh presents massive engineering challenges. Despite matching bandwidth, spreading factors, coding rates, and preambles across both architectures, the LR1121 would consistently drop packets from the Supreme, throwing hardware-level HEADER_ERR interrupts.

The root cause was isolated to Sync-Word Encoding Asymmetry. The SX126x's standard "PRIVATE" 0x12 sync word is encoded asymmetrically on the physical layer (resulting in over-air bytes [0x14, 0x24]). The LR1121's hardware demodulator attempts to read this as a single byte and fails to validate the header.

The Solution: The OMEGA firmware standardizes the entire mesh federation on the LoRaWAN "PUBLIC" 0x34 sync word. Because both the SX126x and LR11x0 families encode this public sync word identically on the physical layer, cross-chip packet drops are completely eliminated, allowing heterogeneous fleets of edge nodes to relay packets flawlessly.

Educational Deep Dive: Sensor Wiring

I2C Bus Capacitance

The I2C protocol was designed for communication between chips on the same circuit board. In marine engineering, we often abuse this protocol by running I2C cables 5 meters down into the ocean. If you use standard 10kΩ pull-up resistors, your sensors will silently fail.

The Core Concept

Parasitic Capacitance ($C_p$)

As a copper wire gets longer, it acts like a capacitor. When the microcontroller tries to release the I2C data line to let it float HIGH (3.3V), the wire's capacitance slowly "charges up" instead of instantly snapping to 3.3V. This rounds off the square wave, causing read errors.

The Physics

The RC Time Constant

The time ($T$) it takes for the voltage to rise is determined by the equation: $T = R \times C_p$. Since we cannot easily lower the capacitance ($C_p$) of a physical 5-meter underwater cable, our only option is to drastically lower the pull-up resistance ($R$).

Step-by-Step Implementation: Calculating Pull-Up Resistors

Problem: You want to run a BME280 sensor on a 5-meter tether using 100kHz standard mode I2C. The maximum allowed rise time ($t_r$) for 100kHz I2C is $1000\text{ns}$.

  1. Estimate Cable Capacitance: A typical shielded twisted pair cable has a capacitance of ~50pF per meter. $$C_p = 5\text{m} \times 50\text{pF/m} = 250\text{pF}$$
  2. Calculate Maximum Resistance ($R_{max}$): Using the standard I2C formula $R_{max} = \frac{t_r}{0.8473 \times C_p}$: $$R_{max} = \frac{1000 \times 10^{-9}}{0.8473 \times 250 \times 10^{-12}} \approx 4721 \Omega$$
  3. Calculate Minimum Resistance ($R_{min}$): The resistor must not allow more than 3mA of sink current ($I_{OL}$) when the chip pulls the line low. Assuming a 3.3V ($V_{cc}$) system with a max $0.4V$ low-level voltage ($V_{OL}$): $$R_{min} = \frac{V_{cc} - V_{OL}}{I_{OL}} = \frac{3.3V - 0.4V}{3\text{mA}} = 966 \Omega$$
  4. Select the Resistor: Your pull-up resistor must be between $966\Omega$ and $4.7\text{k}\Omega$. To ensure crisp square waves against underwater noise, select a strong pull-up near the minimum, such as $2.2\text{k}\Omega$. Do NOT use the default $10\text{k}\Omega$ built into most breakout boards!
Embedded C++

T-Beam Supreme Architecture.

Located in firmware/tbeam_supreme_dual_sensor/, the C++ codebase is highly optimized for the ESP32-S3 microcontroller. The firmware aggressively manages deep-sleep power states, shutting down the GNSS (GPS) module and the Semtech SX1262 LoRa radio between transmission windows.

Core

ESP32-S3

Dual-core XTensa LX7 @ 240MHz. Handles TLV compression and mesh logic.

RF Transceiver

Semtech SX1262

LoRa CSS modulation. Capable of +22dBm output for over-the-horizon links.

Location

u-blox MAX-M10S

Ultra-low-power GNSS. Acquires hot-fixes in <2 seconds before sleeping.

Thermodynamics: A remote sensor floating in the Pacific cannot plug into a wall outlet. The ESP32-S3 draws roughly 100mA while active, draining a standard 18650 lithium battery in two days. By forcing the hardware into the ULP (Ultra Low Power) sleep state, the draw drops to 15µA.

Advanced Encoding

Lossless Predictive Batching.

Transmitting every single temperature sample individually across a low-bandwidth LoRa link introduces massive overhead from packet headers, leading to duty-cycle violations. The OMEGA firmware employs Phase 1 Predictive Batching to completely eliminate this overhead.

The edge node samples its BME280 sensor every 3 seconds into a fast internal ring buffer. Once 10 readings are collected, it encodes them using first-order predictive coding: it transmits the very first reading as a full 32-bit integer, and subsequent readings as tightly packed 4-bit or 8-bit residuals (the delta from the previous reading). This mathematically guarantees a lossless reconstruction of the original high-frequency data array while drastically crushing the packet size.

Network Disruptions

Store & Forward Resilience.

A drifting sensor node will inevitably pass behind a wave or an island, losing contact with the LoRa mesh federation. If the node blindly attempts to transmit, the data is lost forever. To prevent this, the DTO v3.0 firmware implements an autonomous Store & Forward architecture.

When the CSMA-CA (Carrier-Sense Multiple Access with Collision Avoidance) logic detects that the channel is jammed or no acknowledgments are received, the ESP32-S3 halts transmission. The encoded telemetry batches are serialized and dumped directly into the physical NVRAM/SPIFFS flash storage. The node then gracefully returns to deep sleep. Once RF connection is mathematically proven to be re-established (e.g. by receiving a heartbeat from the T-ETH-ELITE gateway or a passing LR1121 relay), the edge node flushes its flash buffer, replaying hours of accumulated telemetry over the mesh.

Edge UI & Telemetry

OLED Display & PMU Diagnostics

Deploying a headless edge node into the ocean is dangerous if field technicians cannot visually confirm its operational state before launch. The latest firmware introduces a robust local UI using an onboard SH1106 OLED screen and direct I2C communication with the AXP2101 Power Management Unit (PMU).

  • Dynamic Field Observability: The OLED dynamically renders the assigned operator_label (e.g. TBSP-S-01), alongside the chip factory eFuse physical_mac directly on the boot screen.
  • Live Hardware Diagnostics: By querying the internal ESP32-S3 die temperature and the AXP2101 PMU, the screen reports real-time battery voltage, charge percentage, and thermal metrics.
  • GPS Rendering: Latitude and longitude coordinates are painted to the display the moment the u-blox GNSS module acquires a hot fix. Hardware variants like the T-Beam 1W and LR1121 are also natively supported with specific flash configurations.
void update_oled_display() {
    display.clear();
    // 1. Render Identity
    display.drawString(0, 8, "OA " + String(operator_label) + "   " + String(uptime));
    
    // 2. Render Hardware Health
    float esp_temp = temperatureRead();
    if (pmu.isReady()) {
        float batt_pct = pmu.getBatteryPercent();
        display.drawString(0, 19, "T:" + String(esp_temp) + "C  bat:" + String(batt_pct) + "%");
    }
    
    // 3. Render Position
    display.drawString(0, 41, String(gps.location.lat(), 4) + "," + String(gps.location.lng(), 4));
    
    display.display();
}
Bare-Metal Realities

Hardware Quirks Compendium.

Working close to the metal inevitably exposes manufacturing anomalies, physical tolerance limits, and undocumented chip behavior. OMEGA's firmware handles these extreme edge cases automatically.

Voltage Fallback

TCXO Initialization

While standard RadioLib drivers default to a 1.6V TCXO logic level, the Supreme boards require exactly 1.8V to initialize the SX1262 oscillator. The firmware implements an automatic fallback cascade (1.8V → 1.6V → XTAL) to prevent fatal radio panics.

Bootloader Limits

4MB vs 8MB Partitioning

The T-Beam 1W hardware ships with only 4MB of flash memory, whereas its binary header mistakenly defaults to 8MB. This mismatch triggers an infinite boot-loop. Custom PlatformIO partition rules (default.csv) explicitly clamp the addressing logic to 4MB.

HPA Clamping

Amplifier Protection

The SX1262 can only internally generate +22 dBm. If a user queries for +30 dBm, the chip will throw an initialization error. The firmware dynamically clamps requests to +22 dBm inside the SX1262, allowing the 1W's external High Power Amplifier (HPA) to cleanly boost the signal over the horizon.
