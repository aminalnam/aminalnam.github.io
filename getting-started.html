Getting Started

A quick-start guide to deploying the OMEGA shore gateway locally and injecting your first synthetic LoRa telemetry packet.

Step 1

Clone and Spin Up.

The OMEGA Shore Gateway is entirely containerized using Docker and Docker Compose. You do not need to install Python, Node, or Redis natively on your host machine to run the platform.

This command pulls the aminalnam/omega repository, builds the FastAPI backend and React frontend, and spins up the SQLite WAL databases and Redis event buses locally.

Terminal 
git clone https://github.com/aminalnam/OMEGA.git
cd OMEGA

# Start the stack in detached mode
docker-compose up -d

# Verify containers are healthy
docker ps
Step 2

Access the Mission Portal.

With the containers running, the FastAPI gateway is exposed on port `8000`, and the React Vite Mission Portal is exposed on port `3000`.

Navigate to http://localhost:3000 in your browser. You will see the Mapbox GL JS canvas, but it will be entirely empty because no hardware nodes are actively transmitting.

Browser Navigation 
Gateway API (Swagger): http://localhost:8000/docs
Mission Portal UI:     http://localhost:3000
Step 3

Inject a Synthetic Payload.

Because you likely do not have a physical SX1262 LoRa modem attached via Serial to your laptop, we provide a mock ingestion endpoint. This simulates the exact byte-level behavior of the gateway-c-daemon passing a frame to the Python router.

Watch your Mission Portal screen; the instant you fire this cURL command, a marker will materialize off the coast of California.

Terminal 
curl -X 'POST' \
  'http://localhost:8000/v1/hardware/mock-rx' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
  "node_id": "urn:omega:node:buoy-alpha",
  "lat": 37.7749,
  "lon": -122.4194,
  "metrics": {
    "water_temp_c": 14.2,
    "battery_v": 4.1
  }
}'
