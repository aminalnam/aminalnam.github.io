Data Pipeline

The Measurement-to-Map Architecture.

Turning acoustic transducer pings into continuous spatial map layers requires a strict pipeline. Edge compute nodes execute physical corrections and heuristic validations in real-time, drastically reducing the data transmitted over the mesh network. Only "accepted" geospatial points reach the Python backend for Inverse Distance Weighting interpolation.

Why process at the edge?

Transmitting raw 10Hz sonar pings over a 250-byte MTU LoRa connection is impossible. By executing sound-speed corrections and strict geographic validation heuristics on the ESP32 rover itself, we only transmit high-value data points. This is the essence of extreme edge-computing in marine environments.

Strict Thresholds

The Validation Matrix.

The ESP32 rover code defines highly specific bounds for data acceptance. If a sensor reading violates any of these predefined thresholds, the data is tagged with an error string and completely excluded from the mapping payload.
