Data Theory

TLV vs JSON Compression.

The JSON envelope above is mathematically huge (~450 bytes) due to structural characters like quotes, spaces, and brackets. If transmitted over LoRa, it would consume dangerous amounts of airtime.

Standard JSON (450 Bytes) 
{"battery-voltage": {"u": "V", "v": 3.92}}

The JSON representation explicitly spells out every key. The string "battery-voltage" alone costs 15 bytes of payload capacity.

TLV Hex (92 Bytes) 
0xa2 0x01 0x82 0xa2 0x02 0x03 0x03 0xfb 0x40 0x0f 0x5c ...

The lora_tlv_bridge.py crushes the string keys into integers (e.g., 1 = battery) and packs the floats into strict IEEE 754 binaries, mathematically reducing the payload by over 75%.

lora_tlv_bridge.py 
import struct

def pack_tlv_battery(voltage: float) -> bytes:
    """
    T = Tag (0x01 for Battery Voltage)
    L = Length (0x04 for 4-byte float)
    V = Value (IEEE 754 32-bit float representation)
    """
    tag = 0x01
    length = 0x04
    # Pack the float 'f' into a big-endian '>' 4-byte binary string
    value = struct.pack('>f', voltage)
    
    return bytes([tag, length]) + value

def unpack_tlv_battery(payload: bytes) -> float:
    # If the tag matches 0x01, extract the 4-byte float
    if payload[0] == 0x01 and payload[1] == 0x04:
        voltage_tuple = struct.unpack('>f', payload[2:6])
        return round(voltage_tuple[0], 2)
    return 0.0

# Example: 3.92V becomes b'\x01\x04@z\xe1H' (Only 6 bytes total!)
binary_frame = pack_tlv_battery(3.92)
