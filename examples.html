Data Theory

TLV vs JSON Compression.

The JSON envelope above is mathematically huge (~450 bytes) due to structural characters like quotes, spaces, and brackets. If transmitted over LoRa, it would consume dangerous amounts of airtime.

Standard JSON (450 Bytes) 
{"battery-voltage": {"u": "V", "v": 3.92}}

The JSON representation explicitly spells out every key. The string "battery-voltage" alone costs 15 bytes of payload capacity.

TLV Hex (92 Bytes) 
0xa2 0x01 0x82 0xa2 0x02 0x03 0x03 0xfb 0x40 0x0f 0x5c ...

The lora_tlv_bridge.py crushes the string keys into integers (e.g., 1 = battery) and packs the floats into strict IEEE 754 binaries, mathematically reducing the payload by over 75%.
