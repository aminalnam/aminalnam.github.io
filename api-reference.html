Authentication Boundary

Security Posture & Headers.

Marine autonomous networks operate in completely untrusted environments. If a malicious actor intercepts the LoRa traffic or scans the public IP of the shore gateway, they cannot be allowed to inject false commands into a remote sensor.

All physical hardware endpoints and critical telemetry streams are protected by a strict Authentication Boundary. The gateway enforces the X-OMEGA-API-Key HTTP header using FastAPI dependency injection.

Operational Security: This boundary ensures that public, unauthenticated users can safely query observational data (like current water temperatures via GET requests), but only authenticated, cryptographically verified operators can interact with the /v1/control/commands namespace, completely preventing the hostile hijacking of multimillion-dollar hardware nodes.

Documentation

OpenAPI Schema Generation.

Because the gateway is built natively on the FastAPI framework, the entire API structure is incredibly robust. The OpenAPI (Swagger) schema is not maintained manually; it is automatically generated and structurally guaranteed by strictly typed Pydantic models in Python.

How to use it: When a downstream integrator or a university partner needs to consume OMEGA data, they simply navigate to the /docs endpoint. They are presented with a living, interactive, mathematically accurate definition of every JSON payload, query parameter, and HTTP status code. If a Python developer changes a field from an integer to a float, the Swagger documentation updates instantly, eliminating API drift entirely.

API Reference

/v1/network/telemetry.

The /v1/network router acts as the central nervous system. The telemetry endpoint does not just dump database rows; it mathematically filters by GeoJSON bounding boxes to prevent massive data egress.

Relevance: By passing bounding parameters, operators retrieve only the telemetry actively generated within a specific operational theater.

cURL Execution: Fetch Bounding Box 
curl -X 'GET' \
  'http://omega.local:8000/v1/network/telemetry?min_lat=36.0&max_lat=37.0&min_lon=-123.0&max_lon=-122.0' \
  -H 'accept: application/json' \
  -H 'X-OMEGA-API-Key: ops-token-123'
API Reference

/v1/control/intents/dispatch.

The /v1/control namespace is strictly guarded by the X-OMEGA-API-Key. This router pushes imperative, physical commands directly into the DTN outbox.

Execution: By dispatching an intent, the gateway handles all the low-level TLV serialization and LoRa CSMA-CA timing, asynchronously firing the configuration to the remote buoy.

cURL Execution: Dispatch Configuration 
curl -X 'POST' \
  'http://omega.local:8000/v1/control/intents/dispatch' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -H 'X-OMEGA-API-Key: ops-token-123' \
  -d '{
  "target_node_id": "urn:omega:node:buoy-alpha",
  "intent_type": "SET_SAMPLING_RATE",
  "parameters": {
    "rate_seconds": 600
  }
}'
API Reference

/v1/science/qc-eval.

The science namespace provides raw, unfiltered access to the gateway's analytical pipeline. Oceanographers do not need to use the visual map portal; they can interact programmatically with the gateway.

Operational Use: A Python script can hit this endpoint with a massive JSON array of raw time-series data. The gateway runs the dataset through the QARTOD engine and returns a fully labeled dataset featuring robust sigma flags.

cURL Execution: QARTOD Quality Control 
curl -X 'POST' \
  'http://omega.local:8000/v1/science/qc-eval' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -H 'X-OMEGA-API-Key: ops-token-123' \
  -d '{
  "metric_id": "water_temperature_celsius",
  "timeseries": [15.2, 15.3, 15.4, 99.9, 15.3]
}'

# Returns:
# {"flags": [1, 1, 1, 4, 1]} # Where '4' indicates a catastrophic sensor failure
Strict Validation

JSON Schema Enforcement.

To protect the core SQLite database from malformed, corrupt, or actively malicious payloads injected by rogue edge nodes, OMEGA relies on strict mathematical JSON schema validations located in the schemas/ directory.

Execution: Before a network packet is ever parsed by the Python backend, it is rapidly validated against the observation-envelope.schema.json definition using the jsonschema library. Any payload missing critical routing arrays, origin UUIDs, or containing the wrong data type is instantly dropped, ensuring database integrity is never compromised by field errors.

observation-envelope.schema.json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://example.invalid/omega-wave/observation-envelope.schema.json",
  "title": "OMEGA-wave Observation Envelope",
  "description": "Canonical application-level envelope for sensor observations.",
  "type": "object",
  "additionalProperties": false,
  "required": [
    "spec",
    "message_id",
    "message_type",
    "origin",
    "routing",
    "sample",
    "payload"
  ]
}
