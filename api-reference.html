Authentication Boundary

Security Posture & Headers.

Marine autonomous networks operate in completely untrusted environments. If a malicious actor intercepts the LoRa traffic or scans the public IP of the shore gateway, they cannot be allowed to inject false waypoints into a remote sensor.

All physical hardware endpoints and critical telemetry streams are protected by a strict Authentication Boundary. The gateway enforces the X-OMEGA-API-Key HTTP header using FastAPI dependency injection.

Operational Security: This boundary ensures that public, unauthenticated users can safely query observational data (like current water temperatures via GET requests), but only authenticated, cryptographically verified operators can interact with the /v1/control/commands namespace, completely preventing the hostile hijacking of multimillion-dollar autonomous assets.

Documentation

OpenAPI Schema Generation.

Because the gateway is built natively on the FastAPI framework, the entire API structure is incredibly robust. The OpenAPI (Swagger) schema is not maintained manually; it is automatically generated and structurally guaranteed by strictly typed Pydantic models in Python.

How to use it: When a downstream integrator or a university partner needs to consume OMEGA data, they simply navigate to the /docs endpoint. They are presented with a living, interactive, mathematically accurate definition of every JSON payload, query parameter, and HTTP status code. If a Python developer changes a field from an integer to a float, the Swagger documentation updates instantly, eliminating API drift entirely.

API Reference

/v1/network Routers.

The main.py API router acts as the central nervous system for the portal UI. It exposes endpoints like /v1/network/remote-access and /v1/network/mesh-topology.

Relevance: These specific endpoints do not return sensor data; they return the live mathematical graph definitions of the RF environment itself. This allows the Mission Portal to draw the exact Signal-to-Noise Ratio (SNR) and RSSI edges between floating relay nodes, allowing the operator to visually identify weak links in the mesh topology before a node drops entirely offline.

API Reference

/v1/control Routers.

The /v1/control/commands namespace is the most dangerous endpoint in the gateway. Strictly guarded by the X-OMEGA-API-Key, this router pushes imperative, physical commands directly into the DTN outbox.

Execution: An operator hitting this API can execute massive remote operations: triggering firmware OTA drops to 50 buoys simultaneously, altering the low-level PID tuning loops of a remote sensor's thrusters, or commanding an underwater modem to transmit a JANUS acoustic payload to a submerged ROV. It is the absolute command-and-control conduit.

API Reference

/v1/science Routers.

The science namespace provides raw, unfiltered access to the gateway's analytical pipeline. Oceanographers do not need to use the visual map portal; they can interact programmatically with the gateway.

Operational Use: A Python script can hit the /v1/science/qc-eval endpoint with a massive CSV batch of raw time-series data. The gateway will run the dataset through the QARTOD engine and return a fully labeled dataset featuring robust sigma flags, flat-line detection parameters, and statistical deviations, providing massive computational power to external researchers.

Strict Validation

JSON Schema Enforcement.

To protect the core SQLite database from malformed, corrupt, or actively malicious payloads injected by rogue edge nodes, OMEGA relies on strict mathematical JSON schema validations located in the schemas/ directory.

Execution: Before a network packet is ever parsed by the Python backend, it is rapidly validated against the observation-envelope.schema.json definition using the jsonschema library. Any payload missing critical routing arrays, origin UUIDs, or containing the wrong data type is instantly dropped, ensuring database integrity is never compromised by field errors.

observation-envelope.schema.json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://example.invalid/omega-wave/observation-envelope.schema.json",
  "title": "OMEGA-wave Observation Envelope",
  "description": "Canonical application-level envelope for sensor observations.",
  "type": "object",
  "additionalProperties": false,
  "required": [
    "spec",
    "message_id",
    "message_type",
    "origin",
    "routing",
    "sample",
    "payload"
  ]
}
