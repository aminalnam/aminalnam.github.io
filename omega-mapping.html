Part 3: Mapping

IDW mapping and overlays.

How OMEGA turns accepted survey points into bathymetric maps, interpolated surfaces, contours, and Google Earth overlays.

Mapping file

Measurement context stays attached.

Every accepted point in OMEGA carries more than a latitude, longitude, and depth. The map file preserves the surrounding measurement conditions.

Representative fields 
point_index,date_utc,time_utc,lat,lng,depth_m,
temp_c,satellites,hdop,speed_kmph,
fix_age_ms,pitch_deg,roll_deg,imu_accuracy
Why this matters

A good map depends on how the point was collected. Satellite count, HDOP, fix age, speed, and orientation explain why a point was accepted.

Interpretation

Filtered before mapping

By the time a record reaches the mapping CSV, it has already passed logging rules for position quality, motion, timing, and depth validity.

Interpolation can make weak data look deceptively smooth. OMEGA reduces that risk by treating validation as part of mapping.

Surface estimation

Inverse distance weighting.

OMEGA's interpolation step creates a regular grid over the surveyed footprint and estimates the depth at each cell using IDW.

From google_earth_overlay.py 
def idw_interpolation(x, y, z, xi, yi, power=2.0, epsilon=1e-12):
    zi = np.zeros_like(xi, dtype=float)
    for row in range(xi.shape[0]):
        for col in range(xi.shape[1]):
            dx = x - xi[row, col]
            dy = y - yi[row, col]
            dist = np.sqrt(dx * dx + dy * dy)
            if np.any(dist < epsilon):
                zi[row, col] = z[np.argmin(dist)]
                continue
            weights = 1.0 / np.power(dist, power)
            zi[row, col] = np.sum(weights * z) / np.sum(weights)
    return zi
What the code is doing

For each cell, it computes a weighted average depth. Close points dominate. Distant points still matter, but less.
