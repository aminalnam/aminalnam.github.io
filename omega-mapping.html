Part 3: Mathematics

Spatial Map Interpolation.

A roving vessel only captures a discrete "breadcrumb trail" of points. To create a 3D topographical map, OMEGA uses Python and Inverse Distance Weighting (IDW) to infer the depth of the unmapped spaces between the lines.

The Void

Inferring the unknown.

Even a dense survey zig-zagging back and forth across a lake leaves vast gaps between the survey lines. We must use a deterministic mathematical model to "guess" the depth of those unmeasured gaps.

The IDW Algorithm

Inverse Distance Weighting

IDW operates on a simple geographic principle: things that are closer together are more related than things that are far apart. To calculate the depth of an unknown pixel on our grid, IDW looks at all the known survey points around it.

It averages their values, but it weights the average based on distance. A survey point 1 meter away has a massive influence on the pixel's estimated depth; a survey point 50 meters away has almost zero influence.

K-D Trees

Solving the N-Body Problem

If we generate a high-resolution 1000x1000 pixel map grid (1,000,000 pixels), and our survey collected 50,000 data points, a naive IDW calculation would require 50 billion distance checks. This would take minutes to compute.

OMEGA solves this using scipy.spatial.cKDTree. A K-Dimensional Tree partitions the data spatially, allowing the algorithm to instantly query the "nearest neighbors" without checking every single point.

Python Implementation

Generating the Contours.

Once the IDW algorithm fills in the grid, we have a continuous 2D array of depth values. OMEGA uses Python's `matplotlib` to render this array into topological contour rings and color-mapped surfaces.

Backend Script 
import numpy as np
from scipy.spatial import cKDTree
import matplotlib.pyplot as plt

def generate_idw_surface(x_coords, y_coords, depth_vals, grid_res=1000):
    # 1. Create the blank target grid
    xi = np.linspace(min(x_coords), max(x_coords), grid_res)
    yi = np.linspace(min(y_coords), max(y_coords), grid_res)
    xi, yi = np.meshgrid(xi, yi)
    grid_points = np.c_[xi.ravel(), yi.ravel()]

    # 2. Build the K-D Tree for lightning-fast spatial queries
    tree = cKDTree(np.c_[x_coords, y_coords])
    
    # 3. Query the 8 nearest neighbors for every pixel in the grid
    distances, indices = tree.query(grid_points, k=8)
    
    # 4. Apply Inverse Distance Weighting (Power = 2)
    weights = 1.0 / (distances ** 2)
    z_estimates = np.sum(weights * depth_vals[indices], axis=1) / np.sum(weights, axis=1)
    
    # 5. Render to Contours
    plt.contourf(xi, yi, z_estimates.reshape((grid_res, grid_res)), cmap='ocean')
    plt.savefig("bathymetry_output.png")
The KML Overlay Export

The resulting PNG file is completely georeferenced. A secondary script binds the corner pixel coordinates of the image to latitude/longitude bounds and wraps them in a KML format, allowing the generated bathymetry map to be instantly dragged and dropped into Google Earth or imported into QGIS.
