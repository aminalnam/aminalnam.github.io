Geospatial data

Bathymetry begins when depth is tied to location.

A single depth value only says how far the bottom was below the sensor at one moment. Bathymetry needs depth plus coordinates so the measurement can become part of a spatial pattern.

Logged structure

The mapping CSV keeps the context attached

The map file records each accepted point with time, latitude, longitude, depth, temperature, satellite count, HDOP, speed, fix age, pitch, roll, and IMU accuracy. That structure keeps the measurement conditions attached to the point that enters the mapping pipeline.

Representative header

point,date_utc,time_utc,lat,lng,depth_cm,temp_c,satellites,hdop,speed_kmph,fix_age_ms,pitch_deg,roll_deg,imu_acc

Two outputs

Raw logging and mapping logging serve different jobs

The raw data log preserves the complete field record, including conditions that may not be suitable for mapping. The map log contains only accepted points. That separation makes the reconstruction step more predictable and keeps the record transparent.

Interpolation

OMEGA uses inverse distance weighting to estimate values between measured points.

The Python mapping scripts build a grid over the survey area and estimate a depth for each grid cell. Nearby measured points receive more weight than distant ones.

From csv_to_kmz_depth_overlay.py 
def idw_interpolation(x, y, z, xi, yi, power=2.0, epsilon=1e-12):
    zi = np.zeros_like(xi, dtype=float)

    for row in range(xi.shape[0]):
        for col in range(xi.shape[1]):
            dx = x - xi[row, col]
            dy = y - yi[row, col]
            dist = np.sqrt(dx * dx + dy * dy)

            if np.any(dist < epsilon):
                zi[row, col] = z[np.argmin(dist)]
                continue

            weights = 1.0 / np.power(dist, power)
            zi[row, col] = np.sum(weights * z) / np.sum(weights)

    return zi
What the algorithm is doing

For each grid cell, the script computes the distance to every accepted survey point. It then gives more influence to closer points by using weights of 1 / distance^power. The epsilon test preserves exact measured points: if the grid lands directly on a survey point, the script returns that measured depth instead of averaging it.

Grid generation

The interpolation only works after the survey area is turned into a computational grid.

The script first finds the bounds of the accepted longitude and latitude values, then uses those bounds to generate a regular mesh of locations where estimated depths will be calculated.

Grid setup 
def build_grid(points, grid_size, power):
    lats = np.array([p["lat"] for p in points], dtype=float)
    lons = np.array([p["lon"] for p in points], dtype=float)
    depths = np.array([p["depth_m"] for p in points], dtype=float)

    lon_min, lon_max = lons.min(), lons.max()
    lat_min, lat_max = lats.min(), lats.max()

    xi_vals = np.linspace(lon_min, lon_max, grid_size)
    yi_vals = np.linspace(lat_min, lat_max, grid_size)
    xi, yi = np.meshgrid(xi_vals, yi_vals)
    zi = idw_interpolation(lons, lats, depths, xi, yi, power=power)

    return zi, lon_min, lon_max, lat_min, lat_max
How the parameters matter

grid_size changes the density of the output surface. A larger grid creates a finer-looking map but increases computation. power controls how quickly distant points lose influence. Higher power makes the surface cling more tightly to nearby points.

Outputs

The same accepted points can become several different map products.

OMEGA’s Python scripts use the same mapping dataset to build different kinds of visual and geographic outputs. Each output highlights a different way of reading the survey.
