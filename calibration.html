Sensor Calibration.

Proving the data is real by mathematically mapping $5 thermistors against NIST-traceable standards.

View OMEGA on GitHub Art Portfolio
The Burden of Proof

NIST Traceability

If you claim the ocean temperature has risen by 0.5°C over a year, the scientific community will immediately ask: "How accurate is your thermometer?" If you cannot prove your $5 DS18B20 thermistor is accurate to a known standard, your data is scientifically useless.

Prior to deployment, every OMEGA probe undergoes a multi-point calibration. The probes are submerged in a controlled water bath alongside a $5,000 reference thermometer that is certified by the National Institute of Standards and Technology (NIST). We record the delta between the cheap probe and the NIST probe at 5°C, 15°C, and 25°C. We then apply a polynomial regression curve in the C++ firmware to perfectly correct the offset, achieving research-grade accuracy on a budget.

Mathematical Calibration Curve 
2nd-Order Polynomial Regression
y = ax² + bx + c

Where:
  y = The True (NIST-traceable) Temperature
  x = The Raw Sensor Reading (DS18B20)
  a = Quadratic Coefficient (Non-linear curvature)
  b = Linear Coefficient (Slope / Gain error)
  c = Constant (Offset / Zero-point error)

Example Calibration Constants derived from the Water Bath:
  a = -0.00014
  b = 1.0023
  c = -0.15

If the raw sensor reads 20.00°C:
  y = (-0.00014 * 400) + (1.0023 * 20.00) + (-0.15)
  y = -0.056 + 20.046 - 0.15
  y = 19.84°C (True NIST-corrected output)
firmware/sensors/calibration.cpp 
// Struct holding EEPROM flashed calibration constants
struct CalibrationConstants {
    float a_quad;
    float b_linear;
    float c_offset;
};

float apply_nist_calibration(float raw_temp, CalibrationConstants cal) {
    // y = ax^2 + bx + c
    float quadratic_term = cal.a_quad * (raw_temp * raw_temp);
    float linear_term = cal.b_linear * raw_temp;
    
    return quadratic_term + linear_term + cal.c_offset;
}
