Quality Review

                
    
                  
  1. Run QC Review — runs QARTOD checks (range, spike, flat-line, stuck-value) on the active metric
    2. 
                  
  2. Read the summary strip — pass / suspect / fail counts
    3. 
                  
  3. Focus or Export — jump the map to the first flag, or export flags as JSON for a notebook
    4. 
                

              

              

                
                
                
              

            

            

              
              
              
            

            

              ▸ 💡 What QARTOD checks for + example workflows
              
    
                
  1. Range test — flags values outside the metric's physical or sensor-spec range (e.g. SST below -2°C or above 40°C).
    2. 
                
  2. Spike test — flags values where the rate of change between samples exceeds a threshold. Common cause: sensor glitch, comms hit.
    3. 
                
  3. Flat-line test — flags identical values across N consecutive samples. Common cause: stuck sensor, frozen DAC.
    4. 
                
  4. Workflow: Pick a metric in Chart Builder → switch to Quality → Run QC Review → 🎯 Focus first flag → map pans to the offending station → Inspect that site → ✎ add an annotation explaining the issue.
    5. 
                
  5. Hand-off: ⤓ Export flags JSON → attach to a Sensor Health ticket → ops team can reproduce + decide whether to swap the unit.
    6.

Bathymetry · OMEGA mapping ROV

                
🌊 Bathymetric Map Workspace

                
Upload an OMEGA-format ROV CSV (or our canonical lat/lon/depth_m shape) and the gateway will ingest every sounding as a bathymetry-depth observation. Generate an IDW-interpolated grid + export as KML/KMZ for Google Earth, or open the layered map view for a live overlay. ROV nodes flashed with firmware/mapping_rov_mega or firmware/mapping_rov_esp32 stream their soundings live into the same store.

              ⬇ Grid KML
              ⬇ Grid KMZ
              ⬇ Points KML
              ⬇ Points CSV

📤 Upload sounding CSV

                
Accepts the OMEGA standalone-logger format (depth_cm, MM/DD/YYYY date, speed_kmph) or the canonical OMEGA-wave shape (depth_m, ISO date, speed_mps). Comment lines starting with # are skipped.

                
                
                
                
                

                  
                

                
Pick a CSV file to enable upload.

              


              

                
🧮 IDW grid parameters

                
The interpolated grid is built on the server from the soundings in measurements. Tune the IDW power + neighbor count to balance smoothness vs. fidelity. The KML/KMZ downloads use these same params.

                
                
                
                
                

                  
                

                
Click Build grid preview to compute + summarize the IDW surface.

              

            


            

              Help — OMEGA CSV format
              
# ROV MAP v1.1
point,date_utc,time_utc,lat,lng,depth_cm,temp_c,satellites,hdop,speed_kmph,fix_age_ms,pitch_deg,roll_deg,imu_acc
0,04/11/2026,17:30:00,34.0122986,-118.4567028,280.3,17.12,7,1.4,1.0,560,-3.7,0.1,2
1,04/11/2026,17:30:05,34.0122967,-118.4566507,258.1,17.17,11,1.1,2.1,180,3.8,1.4,2

              
Required: lat, lon (or lng), depth_m (or depth_cm). Optional: date_utc+time_utc, temp_c, satellites, hdop, pitch_deg, roll_deg, speed_mps (or speed_kmph). Files from aminalnam/OMEGA's standalone Arduino logger are accepted directly.

    6. 
                
  6. "De-clutter." → 🧹 Clear stale watches → removes watches whose baseline is older than 24h (likely deployed sensors that went offline).

            How this page works
            
    
              
  1. Sources fetch automatically when this page opens — NOAA / NOS, marine-tech blogs, ocean-research feeds. The status line under each story shows where it came from.

Login Security

                
Rate-limited IPs

              

              

                
                
              

            

            
Admin-only. After 5+ failed logins against the same username in 10 minutes, the source IP is parked here in cooldown. Subsequent login attempts from that IP get 429'd. Use Unblock to manually clear a row (e.g. legit operator stuck after a typo storm). Clear all is an escape hatch — drops every cooldown. Behind a reverse proxy? Read the proxy setup guide so per-IP attribution works correctly.
