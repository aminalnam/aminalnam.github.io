Machine Learning.

Using historical telemetry to train ARIMA and LSTM models to predict thermal anomalies before they happen.

Predictive Analytics

Time-Series Forecasting

Plotting current temperatures on a map is useful for monitoring, but it is fundamentally reactive. To truly protect an ecosystem from events like coral bleaching, OMEGA must become predictive.

ARIMA Models

Statistical Baselines

For highly seasonal, localized data (e.g., daily tidal heating cycles at a specific reef), we utilize AutoRegressive Integrated Moving Average (ARIMA) models. These statistical models analyze historical telemetry to establish the "normal" sinusoidal rhythm of the ocean, instantly flagging any statistical deviations.

y't = c + φ1y't-1 + ... + φpy't-p + θ1εt-1 + ... + θqεt-q + εt

Where p = autoregressive lag, q = moving average window, and y' = differenced telemetry.

LSTM Neural Nets

Deep Learning

For complex, multi-variate forecasting—such as predicting how wind speed, salinity, and temperature will dynamically interact 48 hours from now—we export data to train Long Short-Term Memory (LSTM) neural networks using TensorFlow. Unlike ARIMA, LSTMs can identify non-linear relationships across massive spatial networks, warning researchers days before anomalies occur.

Network Architecture

LSTM Tensor Flow

Because oceanographic data is sequential, standard feed-forward networks suffer from the vanishing gradient problem. OMEGA uses LSTMs, which maintain an internal "cell state" allowing the network to remember critical multi-day heating trends while forgetting irrelevant short-term noise.

Data Pipeline

Feature Engineering

Raw telemetry cannot be fed directly into an LSTM. OMEGA's backend pipeline automatically normalizes the SQLite time-series data, imputes missing transmission gaps using linear interpolation, and structures the data into overlapping 24-hour windows.

  • Min-Max Scaling: Prevents variables with large ranges (e.g. atmospheric pressure) from dominating smaller variables (e.g. pH).
  • Cyclical Encoding: Time-of-day is encoded using sin() and cos() transformations to preserve the continuous nature of midnight crossing.
  • Tensor Slicing: Data is structured into 3D tensors [samples, timesteps, features].
machine_learning/train_lstm.py 
import tensorflow as tf
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import LSTM, Dense, Dropout

def build_thermal_anomaly_model(input_shape):
    model = Sequential([
        # 1. Recurrent Layer for temporal memory (48-hr context)
        LSTM(64, return_sequences=True, input_shape=input_shape),
        Dropout(0.2), # Prevent overfitting on noisy ocean data
        
        # 2. Deep abstraction layer
        LSTM(32, return_sequences=False),
        Dropout(0.2),
        
        # 3. Output Layer (Predicting T+48 Temperature)
        Dense(16, activation='relu'),
        Dense(1, activation='linear')
    ])
    
    model.compile(optimizer='adam', loss='mse', metrics=['mae'])
    return model
