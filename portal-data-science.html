Data Science & Inspection.

Transforming raw telemetry into actionable intelligence. The Data workspace features a Live Tiles dashboard, outcome-oriented Quick Actions, and an integrated BBS/Data Library.

Live Dashboard

The Inspect Tab.

Operators don't have time to write SQL queries during active deployments. The Data -> Inspect panel was completely rebuilt around a "Live Tiles" viewport-fit dashboard. It polls `v1/measurements/latest` every 5 seconds, displaying the freshest samples in massive, highly legible UI cards.

UI/UX

Stale-State Boundaries

Live Tiles aren't just numbers. They include hardcoded stale-state thresholds. If a GPS reading is older than 30 seconds, or an environmental reading older than 120 seconds, the tile automatically renders an amber warning border. Status is obvious at a glance.

Drill-Down

Modal Queries

Clicking a Live Tile summons a modal with a Custom Query pre-filled for that specific metric. The operator transitions from a high-level visual summary directly into raw tabular data rows in a single click.

Outcome-Oriented

Quick Actions.

The top of the Inspect panel leads with one-click "Quick Actions" to immediately surface high-value insights without manual filtering.

Pre-configured Routings 
// Quick Action routing payloads in portal.js
const quickActions = {
  "My gateway right now": { endpoint: "latest-measurements", limit: 25 },
  "Where are my nodes?":  { endpoint: "stations", format: "geojson" },
  "Map layers registered":{ endpoint: "map-layers", limit: 200 },
  "Bathymetry summary":   { endpoint: "bathymetry-summary", limit: 25 }
};
The BBS & Data Library

Beyond raw metrics, OMEGA-wave acts as a central repository. The BBS (Bulletin Board System) allows operators to pin specific data snapshots, share them across the mesh network, and build an accessible Data Library for post-mission analysis in Pandas or Jupyter.
