The Reality of Sensors

Dirty Data

Physical sensors fail. An analog wire might slightly corrode, changing its resistance. A bird might sit on a thermistor, rapidly heating it. If this raw, dirty data is injected directly into a climate model, it corrupts the entire scientific conclusion.

To prevent this, the OMEGA Python Gateway implements NOAA's QARTOD (Quality Assurance / Quality Control of Real-Time Oceanographic Data) standards. Every single byte of data that arrives via LoRa is mathematically interrogated before it is allowed to enter the SQLite database.

The Mathematics of Trust

QARTOD Implementation

The Python backend utilizes two primary tests to determine if a data point is physically possible.

Test 1

Gross Range Test

This is a fundamental absolute bound. Liquid ocean water physically cannot exist at 150°C. If the C++ logic board encounters a short circuit and transmits a `temp: 150.0` payload, the Python Gateway's Gross Range Test instantly flags it with a `QARTOD=4` (Fail) code. The data is still logged for hardware debugging, but it is stripped from the React visualization.

Test 2

Rate of Change Test

Ocean water has massive thermal inertia. It takes enormous energy to heat a cubic kilometer of water. If a sensor reports 15°C at 12:00 PM, and 25°C at 12:01 PM, the temperature is technically within the Gross Range, but the rate of change (+10°C in 60 seconds) violates the laws of thermodynamics. The gateway flags this spike as `QARTOD=3` (Suspect).

gateway/services/qartod.py 
def evaluate_gross_range(value: float, sensor_type: str) -> int:
    """NOAA QARTOD Test 1: Gross Range"""
    bounds = QARTOD_THRESHOLDS.get(sensor_type)
    if not bounds:
        return 2 # Not Evaluated
        
    if value < bounds.user_min or value > bounds.user_max:
        return 4 # Fail (Hard Limit)
        
    if value < bounds.sensor_min or value > bounds.sensor_max:
        return 3 # Suspect (Soft Limit)
        
    return 1 # Pass
    
def evaluate_rate_of_change(current: float, previous: float, delta_t_seconds: int) -> int:
    """NOAA QARTOD Test 2: Rate of Change"""
    rate = abs(current - previous) / delta_t_seconds
    if rate > MAX_PHYSICAL_RATE:
        return 4 # Fail: Thermodynamically impossible
    if rate > SUSPECT_RATE:
        return 3 # Suspect: Highly anomalous spike
    return 1 # Pass
Flagging Standards

The QARTOD Flags

OMEGA utilizes the standard 1-9 numbering system mandated by the US Integrated Ocean Observing System (IOOS).

Flag Meaning Action by API
1 Pass Data pushed to WebSockets & External APIs
2 Not Evaluated Logged, but hidden from science dashboards
3 Suspect / Interesting Pushed to UI, highlighted in yellow for manual review
4 Fail Logged for debugging only. Stripped from API payloads.
