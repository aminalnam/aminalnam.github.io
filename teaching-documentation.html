System Bringup

Getting the Gateway Online.

Configuring a local development environment for a system that spans React, FastAPI, SQLite, and LoRa DTN can be daunting. The OMEGA repository utilizes a master PowerShell script omega_gateway_online.ps1 (and its bash equivalent) to orchestrate the entire bringup process.

How to use it: Running this script automatically handles environment variable injection, PYTHONPATH resolution, database initialization, and launches the gateway via uvicorn gateway.main:app --host 0.0.0.0.

Crucially, it actively handles binding the application to the physical network interface card (NIC), exposing the Gateway APIs and the Mission Portal to devices on the same Wi-Fi LAN. It also optionally hooks into daemonized tunneling services like ngrok or cloudflared, punching a secure reverse TCP tunnel out to the internet so remote oceanographic teams can monitor a gateway running on a boat thousand of miles away.

Field Hardware

Raspberry Pi Gateway Mesh.

The ultimate goal of OMEGA is remote, unassisted field deployment. While testing on a Windows laptop is convenient, the physical gateway is usually a headless Raspberry Pi 4 bolted inside a pelican case on a bouncing vessel.

We provide exhaustive deployment guides on configuring the Pi for the harsh realities of marine deployments. This includes setting up rigid systemd daemon services that automatically restart the Python application if it crashes, configuring the OS to mount root filesystems as Read-Only to prevent catastrophic SD card corruption during abrupt power losses, and ensuring the SQLite database WAL files are cleanly flushed to disk.

Software Architecture

The Asynchronous Gateway.

Why do we use FastAPI and asyncio instead of Flask? When 50 nodes transmit LoRa packets at the exact same moment, the gateway must handle severe concurrency without locking up.

ASGI Event Loop

Non-Blocking I/O

Python's Global Interpreter Lock (GIL) prevents true multi-threading. By using await, the main thread can instantly pause a database write, ingest an urgent radio packet, and resume the write, preventing dropped telemetry.

Type Hinting

Pydantic Schemas

FastAPI relies on rigid Pydantic models. If a corrupted, half-transmitted RF packet hits the API, Pydantic throws an HTTP 422 immediately, preventing malformed JSON from crashing the router.

Hardware Limits

Thread Starvation

If we used synchronous threads (like Flask) on a Raspberry Pi, 50 simultaneous connections would starve the CPU context switcher. Async tasks use negligible RAM, easily scaling to thousands of connections.

Database Theory

SQLite WAL Concurrency.

A standard SQLite database locks the entire file during a write transaction. If 50 nodes dump telemetry simultaneously, the database will throw a database is locked exception. OMEGA configures extreme SQLite PRAGMAs to bypass this.

gateway/database.py PRAGMAs 
-- 1. Enable Write-Ahead Logging (WAL)
-- This allows simultaneous Readers and one Writer by appending to a -wal file.
PRAGMA journal_mode = WAL;

-- 2. Drop synchronous mode from FULL to NORMAL
-- The Pi SD card is incredibly slow. We accept a tiny risk of data loss on sudden power failure to gain a 10x write speed boost.
PRAGMA synchronous = NORMAL;

-- 3. Increase memory-mapped cache
-- Allow SQLite to use 64MB of RAM before flushing to the slow SD card.
PRAGMA mmap_size = 67108864;
Frontend Theory

React & The DOM.

When a WebSocket pumps 10 incoming telemetry packets per second into the Mission Portal, a naive React app will re-render the entire DOM 10 times a second, freezing the user's browser.

State Hydration

React Context

The WebSocket listener dumps raw JSON directly into a React Context provider. Only specific components (like the voltage dial) subscribe to that specific slice of state, isolating the re-renders.

Debouncing

Throttle Hooks

If a node dumps a backlog of 500 queued points, the React useThrottle hook mathematically limits the visual map updates to 1 FPS, preventing the GPU from locking up while processing the GeoJSON.
