Edge Compute Physics

The Edge-to-Cloud Continuum.

Marine deployments operate in environments uniquely hostile to traditional IP networks. The physics of transmitting large data volumes over satellite (Iridium/Starlink) or VHF radio links dictates prohibitive costs and restrictive latency boundaries. For instance, streaming raw 20Hz vibration data from a remote buoy to the cloud is economically and physically unviable.

OMEGA bypasses these physical limitations by pushing computational complexity entirely to the edge. Instead of piping raw, high-frequency matrices across the ocean, an onboard NVIDIA Jetson Orin NX or a multi-core ESP32 locally ingests, filters, and reduces millions of data points into a single semantic "Mission Intent" or "Observation Envelope." This process—often involving embedded machine learning or fast Fourier transforms—saves up to 99% of the required bandwidth.

In practice, this means an Autonomous Surface Vehicle (ASV) can detect an acoustic anomaly, classify it locally, and transmit a 450-byte CBOR packet via LoRa to a shore gateway, rather than attempting to transmit a 50MB audio file. This paradigm shift enables massive, scalable sensor networks that can operate continuously without cellular dependencies.

System Architecture

The Tripartite Network.

The OMEGA architecture is strictly divided into three distinct, decoupled strata to enforce fault tolerance and maintain rigid boundaries between hardware execution and human interaction:

1. Physical Sensors & Nodes: This layer comprises the raw hardware—ESP32 microcontrollers, Pixhawk flight controllers, and physical probes. These devices acquire analog/digital signals, manage RTOS tasks, and communicate via mesh protocols like LoRa or HaLow. They have no concept of "the internet" or "user interfaces."

2. The Gateway Normalizers: Python-based software routers (typically running on Raspberry Pi 4s or shore-based servers) act as the vital bridge. They ingest the disjointed, highly compressed binary formats (CBOR/Protobuf) generated by the edge nodes and "inflate" them. The gateway normalizes these packets into unified, queryable SQLite records.

3. The Mission Portal: The React/WebGL interface consumed by human operators. The portal is entirely decoupled from the hardware layer; it interacts exclusively with the Gateway APIs via standard HTTP/REST and WebSockets.

This tripartite division ensures that if the portal server goes offline, the gateway safely continues buffering telemetry into its WAL-enabled database. Conversely, if the gateway dies, the edge node continues to operate safely in its autonomous loop without crashing due to network timeouts.

Failure Modeling

Resilient Architectures.

Marine sensors exist in a state of continuous degradation. Batteries drain unexpectedly in cold water, thermoclines bend and absorb acoustic communications, and salt aggressively corrodes IP67 seals. A brittle, tightly coupled architecture will inevitably collapse when a single critical node vanishes from the network.

OMEGA is fundamentally designed around the principle of Asynchronous Resiliency and Delay Tolerant Networking (DTN). There are no blocking network calls in the core firmware that would freeze an execution thread if an acknowledgment packet is lost in the noise floor.

Instead, data is stored locally in circular flash buffers and forwarded only when a viable physical route—such as a passing ASV acting as a data mule, or an active satellite pass overhead—is mathematically proven to exist. This "store-and-forward" resilience ensures that brief connectivity windows are fully utilized to offload critical envelopes, while preserving battery life during extended blackout periods.
