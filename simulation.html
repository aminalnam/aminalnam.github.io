Synthetic Generation

Procedural Time-Step Environments.

Deploying a massive mesh network without testing is catastrophic. The simulator.py module spawns thousands of virtual "Digital Twins" that run the exact same networking logic as the physical edge nodes.

Procedural Generation: The Python engine dynamically generates synthetic nodes that mathematically drift according to fluid dynamics models. Each tick of the event loop evaluates the Haversine distance and RSSI dropoff between all virtual nodes. If Node A drifts out of range of Node B, the simulator physically severs the virtual connection, forcing the Mesh Core to dynamically recalculate its Delay Tolerant routing tables. This allows developers to stress-test the FastAPI event loop with 10,000 concurrent CBOR packets before touching actual hardware.

gateway/simulator.py
async def tick_digital_twins(twins: list[DigitalTwin], mesh_router: MeshRouter):
    # Simulate kinematic drift and evaluate RF topology.
    for node in twins:
        # Apply fluid dynamic drift (currents & wind)
        node.apply_kinematic_drift()
        
        # Evaluate Line-of-Sight RF range against all other nodes
        visible_peers = []
        for peer in twins:
            distance_km = haversine(node.lat, node.lon, peer.lat, peer.lon)
            if distance_km < LORA_MAX_RANGE_KM:
                visible_peers.append(peer.uuid)
                
        # Update the virtual routing table
        await mesh_router.update_topology(node.uuid, visible_peers)
Network Planning

Monte Carlo Relay Drop Analysis.

Deploying physical hardware into the ocean is incredibly expensive. Before a remote sensor physically drops a floating LoRa relay buoy into the water to extend the mesh network, the operational team must mathematically prove the drop zone is viable.

The simulation harness runs thousands of Monte Carlo physics models. It ingests historical Copernicus wind and tidal current datasets, and simulates the physical drift of the unpowered relay buoy over 72 hours. It calculates the exact GPS coordinate to drop the relay, maximizing the statistical probability that the drifting relay will remain within RF line-of-sight (approx 15km) of both the shore gateway and the moving edge node for the duration of the mission.
