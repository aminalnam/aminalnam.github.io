Fluid Dynamics & Vector Math

Course Over Ground (COG).

An Remote Edge Node (edge node) operates on the surface of a constantly moving, chaotic fluid. If a traditional routing algorithm points the sensor's bow directly at a target (its Heading), but a strong 3-knot cross-current pushes against the hull, the vessel's actual physical path over the Earth (its Course-Over-Ground, or COG) will drift significantly off-target. In tight coastal environments, this results in the edge node crashing into a reef.

The OMEGA Wayfinder module is a high-performance kinematic engine. It models the vessel's kinematics as a physical vector, and imports real-time ocean current and wind data (via NOAA ERDDAP) as an environmental vector field. The engine mathematically calculates the necessary "crab angle"—forcing the edge node to steer into the current so its resulting COG exactly matches the desired track line.

Operational Use: When an operator drops a polygon in the Mission Portal and clicks "Generate Coverage Path," the Wayfinder engine doesn't just draw straight lines. It simulates the fluid dynamics of the specific timestamped mission, ensuring the resulting waypoints actually account for the physical ocean environment.

Algorithm Design

A* Routing on Dynamic Vectors.

Standard terrestrial routing algorithms (like Google Maps) assume static roads with fixed speed limits. Marine routing requires navigating a completely dynamic, continuously shifting fluid field where the "cost" of moving from Point A to Point B changes depending on the direction of the tide.

OMEGA implements adapted A* algorithms operating over a time-stepped geometric grid. The algorithm assigns extreme traversal penalties to nodes featuring strong counter-currents (which would drain the battery rapidly) and rewards nodes featuring following currents (which allow the edge node to surf for free).

How it works: The resulting path often looks bizarre to a human operator—it might curve wildly to avoid an invisible eddy—but it mathematically optimizes for total fuel efficiency rather than strict linear distance, allowing small solar-powered edge nodes to cross massive oceanic basins without draining their lithium reserves.

Wayfinding Execution

Kinematic Route Generation.

The wayfinding-plan.json payload represents the highest level of autonomous intent. A human operator submits this structure to the Gateway, defining abstract polygon bounds and target coordinates.

Execution: The backend pathing engine intercepts this payload and spins up multiple parallel fluid dynamic simulations. Utilizing the speeds_knots and route_offsets_nm arrays, the engine scores thousands of potential paths against live wind and wave vector fields extracted from the Copernicus database. The final output is an optimized string of GPS waypoints that surf the local currents, maximizing the sensor's fuel efficiency.

Coordinate Autonomy

edge node Waypoint Missions.

Remote Edge Nodes (edge nodes) cannot rely on a human operator holding a joystick; they must execute strict mathematical coordinate sequences completely autonomously. The Mission Portal builds strict JSON payloads defining complex bounding geofences, precise transit speeds (in meters per second), and targeted camera polling rates.

Operational Use: An operator draws a polygon over a coral reef in the Mapbox UI. The gateway pathing engine converts that polygon into a lawnmower survey grid of GPS waypoints. This payload is transmitted to the edge node, which then blindly executes the grid, capturing bathymetry data at exactly 1-meter intervals, completely unassisted.

Hydrodynamics

HFRNet Surface Currents.

Attempting to navigate a small, solar-powered edge node directly against a 4-knot Gulf Stream current will drain its lithium battery instantly, leading to a dead vessel adrift at sea. The hfrnet.py module solves this by ingesting the U.S. High-Frequency Radar Network (HFRNet) data via THREDDS servers.

How it works: This provides a real-time, high-resolution vector field of ocean surface currents. The wayfinding.py module consumes this massive vector matrix and executes its A* pathfinding algorithm. The resulting mission plan actively "surfs" the currents, intentionally deviating from a straight line to hitch a ride on localized eddies, drastically extending the sensor's operational range.

Autonomous Safety

Active Guard Geofencing.

The active_guard.py module acts as a merciless, unblinking watchdog over the physical fleet. During high winds or heavy chop, a remote sensor might struggle to maintain its commanded heading and slowly drift off-course.

Execution: Active Guard constantly calculates the cross-track error (the perpendicular distance from the planned path) of every edge node against predefined GeoJSON exclusion zones. If a vessel mathematically deviates into a dangerous shipping lane or a shallow reef, Active Guard instantly overrides the standard waypoint mission, triggering an emergency acoustic beacon to alert recovery vessels before the node is lost.

gateway/active_guard.py
async def check_geofence_breach(node_id: str, current_lat: float, current_lon: float):
    # 1. Load active GeoJSON exclusion polygons (e.g. shipping lanes, shallow reefs)
    exclusion_zones = await db.get_active_exclusion_zones()
    
    # 2. Ray-casting algorithm to determine if point is inside polygon
    for zone in exclusion_zones:
        if point_in_polygon((current_lat, current_lon), zone.polygon):
            logger.critical(f"BREACH: Node {node_id} entered exclusion zone {zone.name}!")
            
            # 3. Fire asynchronous webhook to Mission Portal UI
            await websocket_manager.broadcast_alert("geofence_breach", node_id)
            
            # 4. Mathematically generate orthogonal escape vector
            escape_vector = calculate_orthogonal_escape(current_lat, current_lon, zone)
            await force_override_waypoint(node_id, escape_vector)
            return
