Testing Framework

BETA Framework.

The Build Environment for Testing & Analytics (BETA). A standalone deterministic evaluation engine that continuously subjects backend APIs and gateways to high-frequency synthetic telemetry, measuring pipeline latency, detecting performance regressions, and generating AI-driven remediation tickets.

View OMEGA Integration View Test Suite
The Methodology

Evidence-Based Engineering.

When building a complex system that sits between unpredictable physical edge hardware and high-frequency backend databases, "it works on my machine" is an unacceptable standard. Every commit pushed to the target gateway must be empirically proven to handle chaotic telemetry floods without locking the SQLite WAL or dropping HTTP connections.

Telemetry Generator

Deterministic Fuzzing

BETA generates thousands of synthetic payload injections—simulating multiple drifting devices, acoustic anomalies, and deliberate malformed hardware packets.

Performance Matrices

Tracking Regressions

The system tracks the P95 Query Latency and Ingest Throughput over time. If a new software build drops the ingestion rate below 250 obs/sec, a regression flag is thrown.

Automated Remediation

AI Copilot Ticketing

Post-run, the Honu AI Agent analyzes the performance deltas. If a regression is detected, it automatically writes a Jira-style technical ticket outlining the suspected bottleneck.

Core Component

The Synthetic Injector.

At the heart of the BETA framework is the Synthetic Injector. Written in asynchronous Python, it precisely mimics the behavior of dozens of physical edge nodes deployed in the field. It generates cryptographically signed, CBOR-encoded telemetry payloads and fires them at the Gateway's ingress endpoints.

This allows us to simulate massive telemetry floods (e.g., a sudden sensor surge after an underwater acoustic event) and ensure the SQLite WAL layer and event bus can handle the throughput without locking.

synth_injector.py 
import asyncio
from core.fuzzer import CborPayloadGenerator
from network.client import AsyncGatewayClient

async def run_synthetic_flood(target_url: str, duration_sec: int, tps: int):
    """Fires a sustained flood of synthetic marine telemetry."""
    client = AsyncGatewayClient(target_url)
    generator = CborPayloadGenerator(node_count=50)
    
    print(f"🌊 Initiating flood: {tps} TPS for {duration_sec}s")
    
    end_time = asyncio.get_event_loop().time() + duration_sec
    while asyncio.get_event_loop().time() < end_time:
        payload = generator.create_drift_telemetry()
        # Fire-and-forget payload injection
        asyncio.create_task(client.post_telemetry(payload))
        await asyncio.sleep(1.0 / tps)
        
    await client.close()
    return client.get_latency_metrics()
Performance Thresholds

Analytics & SLIs.

BETA does not just check if a request returns a 200 OK. It measures the deeply embedded Service Level Indicators (SLIs) of the architecture. If a new software build causes a statistical regression in ingestion latency, the build fails.

Automated Remediation

Honu AI Integration.

When the Synthetic Injector flags a performance regression, it doesn't just stop at a red build light. The failure matrices (including CPU profiles, database wait times, and traceback logs) are packaged and forwarded to the Honu AI Copilot.

The Copilot acts as a senior systems engineer. It correlates the current performance drop with the recent git commit diff, hypothesizes the root cause, and automatically drafts a Jira-style remediation ticket outlining the "Next Moves."

⚠️ AI Auto-Triage Ticket Generated
Title: [REGRESSION] P95 Latency spike to 120ms during concurrent SQLite writes
Hypothesis:
The recent commit (fc92b4) introduced an asynchronous index update on the `telemetry_data` table. Because the database is in WAL mode, the prolonged index rebuild is starving the synchronous writer pool, leading to connection timeouts from the Synthetic Injector.

Recommended Remediation:
1. Move the index update to a background worker queue (e.g., Celery) rather than blocking the main FastAPI thread.
2. Ensure `PRAGMA synchronous = NORMAL` is strictly enforced during high-throughput ingestion windows.
