Part 1: Physics

Acoustics & Thermodynamics.

A transducer does not measure depth; it measures time. This chapter explores the thermodynamic math required to convert acoustic time-of-flight into an accurate geographic spatial anchor.

Back to Overview Next: Validation Logic
Acoustic Fundamentals

The Time-of-Flight Equation.

The core mechanism of any active sonar system is Time-of-Flight (ToF). A piezo-electric transducer physically strikes the water, emitting an acoustic pulse. It then acts as a microphone, listening for the echo rebounding off the substrate.

Mathematical Model

Solving for Z

The time recorded by the microcontroller is the *round-trip* duration. To find the one-way depth (Z), we must divide by two and multiply by the acoustic velocity of the medium.

Equation

Z = (v × t) / 2

Where Z is Depth, v is the speed of sound in water, and t is the total travel time.

The Hidden Variable

The Problem with Velocity

In standard hobbyist electronics, the speed of sound in air is relatively stable. In marine environments, however, the velocity v is a highly dynamic variable that shifts based on the thermodynamic state of the water column.

Applied Physics

The Mackenzie Equation.

To accurately solve for depth, OMEGA must continuously recalculate the speed of sound. The industry standard for shallow-water acoustics is the nine-term Mackenzie equation, which calculates velocity based on Temperature, Salinity, and Depth/Pressure.

Thermodynamic Sensitivity

Why Temperature Dominates

Of the three variables in the Mackenzie equation, temperature has the most volatile impact in shallow coastal waters. A change from 10°C to 20°C can shift the speed of sound by over 30 meters per second. Over a 50-meter depth ping, failing to correct for this temperature swing results in a massive topological error.

Hardware Architecture

To solve this, OMEGA autonomous rovers are equipped with a waterproof DS18B20 digital temperature probe mounted directly adjacent to the sonar transducer. This allows the edge-compute node to capture a synchronized thermal reading at the exact moment the ping is fired.

Code Walkthrough

Real-Time C++ Correction.

Because full Mackenzie calculation is computationally expensive for an edge microcontroller, OMEGA uses a linearized heuristic correction centered around a baseline calibration.

Arduino Edge Logic 
// Baseline reference values used by the OMEGA edge logger.
const float sonarCalSoundSpeed = 1500.0f;  // m/s (approximate fresh water at 20C)
const float sonarCalTempC = 20.0f;         // Celsius

// Function: Linear approximation of velocity shift based on temp delta
float estimateSoundSpeed(float waterTempC) {
  float c = sonarCalSoundSpeed;
  if (!isnan(waterTempC)) {
    // For every 1 degree C shift, sound speed changes by approx 4.6 m/s
    c += 4.6f * (waterTempC - sonarCalTempC);
  }
  return c;
}

// Function: Rescale the raw hardware output
float correctSonarRangeCm(float reportedRangeCm, float waterTempC) {
  float cActual = estimateSoundSpeed(waterTempC);
  // Apply the ratio of Actual Velocity vs Calibrated Velocity
  return reportedRangeCm * (cActual / sonarCalSoundSpeed);
}
Algorithm Breakdown

Instead of sending raw time-of-flight data over the mesh network (which would require the server to do the math), the ESP32 calculates the velocity ratio (cActual / sonarCalSoundSpeed) locally. It applies this scalar to the hardware's raw distance output, generating an environmentally-corrected Depth (Z) before the packet even leaves the boat.
