Part 1: Sensing

Sensing, sound speed, and depth.

A closer look at how OMEGA estimates underwater depth, why water temperature changes that estimate, and why salinity matters.

Back to Overview Next: Validation
Acoustic measurement

Depth is being inferred from sound.

The sonar transducer sends a pulse, listens for the return from the bottom, and estimates distance from travel time.

Core equation

Round-trip time cut in half

The travel-time measurement includes the trip down and back. The one-way depth formula is:

Formula

depth = (sound speed × travel time) / 2

Why it changes

Speed depends on water

Temperature, salinity, and pressure all change how quickly sound moves through water, changing the interpretation of the depth reading.

Implementation

Temperature-based correction.

In the Arduino code, the sonar uses a baseline reference speed of 1500 m/s at 20 °C, nudged up or down based on measured water temperature.

From rov_logger_mapping.ino 
// Baseline reference values used by the logger.
const float sonarCalSoundSpeed = 1500.0f;  // m/s
const float sonarCalTempC = 20.0f;         // °C

// Estimate sound speed from water temperature.
float estimateSoundSpeed(float waterTempC) {
  float c = sonarCalSoundSpeed;
  if (!isnan(waterTempC)) {
    c += 4.6f * (waterTempC - sonarCalTempC);
  }
  return c;
}

// Rescale the sonar-reported range
float correctSonarRangeCm(float reportedRangeCm, float waterTempC) {
  float cActual = estimateSoundSpeed(waterTempC);
  return reportedRangeCm * (cActual / sonarCalSoundSpeed);
}
What the code is doing

The logger starts from a reference calibration, then adjusts the sound speed based on the DS18B20 water temperature reading.

Depth entry

Corrected before accepted.

The sonar value doesn't go straight into the map. It is corrected, smoothed, and bounds-checked first.

Sonar read loop 
// Convert the decoded sonar packet to centimeters.
float reportedDepthCm = raw / 10.0f;

// Correct that range using the measured water temperature.
float correctedDepthCm = correctSonarRangeCm(reportedDepthCm, tempC);

// Ignore values outside the operating depth window.
if (correctedDepthCm >= 2.0f && correctedDepthCm <= 800.0f) {
  addDepthSample(correctedDepthCm);
  lastValidDepthMillis = currentMillis;
  validReading = true;
}
Why this stage matters

The measurement pipeline does physics and quality control before the mapping logic ever sees the point.
