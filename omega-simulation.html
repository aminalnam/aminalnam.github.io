Synthetic Datasets & Mesh State

Simulation Harness & Digital Twins.

Testing a disconnected DTN mesh network is impossible with a single static JSON file. OMEGA-wave includes a massive Simulation Harness that procedurally generates stationary water-quality buoys, moving drifters, ROV tracks, and JANUS acoustic activity across defined time steps.

Furthermore, the edge nodes are modeled as Digital Twins. Because a node may go offline for days, the portal maintains a decoupled state model (validate_sync_request). The twin tracks variables across states like pending, synced, and drift, ensuring the operator knows whether a hardware parameter shown in the UI has actually reached the physical node.

Time-Step Procedural Generation

The simulator can fast-forward days of simulated time, seeding measurements with realistic statistical noise and drift patterns to stress-test the UI.

ALLOWED_TWIN_STATUSES = {"pending", "synced", "drift", "reported", "failed"}
def validate_sync_request(payload: dict[str, Any]) -> dict[str, Any]:
    variables = payload.get("variables")
    for item in variables:
        status = _optional_string(item, "status")
        if status not in ALLOWED_TWIN_STATUSES:
            raise DigitalTwinValidationError(f"variables.status must be one of {ALLOWED_TWIN_STATUSES}")
