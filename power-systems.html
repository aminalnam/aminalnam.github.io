Energy Acquisition

Maximum Power Point Tracking (MPPT)

Extracting energy from a solar panel on a constantly bobbing ocean buoy requires dynamic electrical load balancing to prevent voltage collapse.

The Problem

Dynamic Impedance

Solar panels are not ideal voltage sources. As a buoy crests a wave and pitches into shadow, the panel's internal impedance wildly fluctuates. If the charging circuit attempts to draw a static 500mA of current, the panel's voltage will instantly collapse below the 3.3V logic threshold, resulting in zero power transfer and a potential brownout.

The Solution

Dynamic Buck-Boost ICs

OMEGA hardware utilizes dedicated MPPT ICs (like the CN3791 or BQ25895) acting as buck-boost DC-DC converters. These chips constantly sample the incoming solar voltage at thousands of Hz. If the voltage begins to dip, the chip dynamically reduces the current draw, riding the exact crest of the P=I*V power curve to extract maximum wattage regardless of cloud cover or wave pitch.

MPPT Algorithm (Perturb and Observe) 
float current_power = get_solar_voltage() * get_solar_current();
if (current_power > previous_power) {
    if (voltage > previous_voltage) {
        increase_duty_cycle(); // Keep climbing the curve
    } else {
        decrease_duty_cycle();
    }
} else {
    if (voltage > previous_voltage) {
        decrease_duty_cycle(); // We passed the peak, back up
    } else {
        increase_duty_cycle();
    }
}
Energy Storage

18650 Thermodynamics

Standard Lithium-Ion batteries fail catastrophically in freezing North Atlantic waters.

The Cold Weather Threat: When standard 18650 Li-ion cells drop below 0°C (32°F), the internal liquid electrolyte begins to solidify. If a solar panel attempts to push a high charge current into a freezing battery, the lithium ions cannot intercalate into the carbon anode. Instead, they form permanent, metallic lithium dendrites. Over time, these dendrites pierce the internal separator, causing a hard short-circuit and catastrophic failure.

The OMEGA BMS: The hardware incorporates a strict NTC thermistor near the battery pack. The firmware prevents any charging current when the ambient temperature approaches freezing, relying solely on solar-direct bypass to power the logic board, ensuring the battery pack survives the winter deployments.

Power Budgeting

The 15µA Sleep State

An ESP32 idling at 40mA will drain a 3000mAh battery in 3 days. To survive a 6-month deployment, extreme power budgeting is enforced.

Deep Sleep

RTC Coprocessor

The primary dual-core 240MHz CPU is completely powered down. Only the Ultra-Low Power (ULP) RTC coprocessor remains active, dropping the entire board consumption to an incredible 15 micro-amps.

Wake Interrupt

Timer & LoRa DIO

The RTC timer wakes the main CPU exactly every 15 minutes to sample sensors. Alternatively, the LoRa radio's DIO pin acts as an external hardware interrupt, waking the CPU only if a valid packet arrives.

TX Burst

1 Amp Spike

When transmitting at +22dBm, the radio briefly spikes to nearly 1,000mA. The hardware utilizes large bypass capacitors to buffer this instantaneous draw and protect the battery from voltage sag.

Resilience

Hardware Watchdogs (WDT)

In the middle of the Pacific Ocean, there is no "reset button". If the firmware enters an infinite loop, the node is lost forever.

Execution: The ESP32 is equipped with an independent, silicon-level Hardware Watchdog Timer. The main Python/C++ loop is required to "pet the dog" (reset the timer) every 5 seconds. If the software crashes, hangs on a bad I2C sensor read, or enters a race condition, it fails to pet the dog. Once the hardware timer expires, the silicon physically cuts power to the CPU and forces a hard reboot, guaranteeing autonomous recovery from any software fault.
