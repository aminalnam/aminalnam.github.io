WebGL Rendering & Spatial Indexing

Mapbox GL JS Performance.

Rendering thousands of live nodes, historical telemetry paths, and dynamic vector polygons simultaneously requires a hardware-accelerated pipeline. Standard DOM manipulation (like traditional Leaflet maps) completely freezes the browser when node counts exceed a few thousand.

The Mission Portal utilizes Mapbox GL JS, pushing the heavy spatial rendering algorithms directly to the client's GPU via WebGL. To handle extreme density deployments, the system clusters grouped nodes using rapid Supercluster algorithms. Rather than manipulating thousands of `

` tags, millions of points are mathematically projected and drawn directly to a single WebGL canvas context.

Offline Capabilities: To save highly constrained bandwidth when an operator is accessing the portal from a ship in the middle of the ocean, the frontend aggressively caches Mapbox Vector Tiles (MVT) in the browser's IndexedDB. This ensures the map loads instantly from local storage, without requiring external API calls over the satellite link.

Network Architecture

HTTP Long-Polling vs WebSockets.

WebSockets are the industry standard for real-time, bidirectional communication, but they carry a fatal flaw for marine operations: they require a persistently open, stable TCP connection. Over degraded satellite networks or maritime VHF links, persistent TCP connections drop constantly, triggering an endless, expensive loop of handshake reconnects.

OMEGA completely bypasses this architectural flaw by utilizing a strict, lightweight HTTP Long-Polling architecture for its dashboard UI. The browser requests state updates via stateless `GET` requests using `async/await` fetch loops.

How it works: If the connection drops mid-poll—which happens constantly in the field—the browser simply catches the timeout error and attempts the next stateless request. There are no dropped connection pipelines to rebuild. This makes the UI incredibly resilient to spotty, high-packet-loss internet.

web/src/hooks/useLongPoll.ts
export function useLongPoll(endpoint: string) {
  useEffect(() => {
    let isActive = true;
    
    const poll = async () => {
      while (isActive) {
        try {
          // Timeout set to 30s. If server has no data, it holds request until timeout.
          const response = await fetch(endpoint, { signal: AbortSignal.timeout(30000) });
          if (response.ok) {
            const data = await response.json();
            dispatch(updateFleetState(data));
          }
        } catch (error) {
          // Silent catch on AbortError (timeout) or network drop.
          // The loop simply restarts instantly without breaking the UI.
          await new Promise(resolve => setTimeout(resolve, 1000));
        }
      }
    };
    poll();
    
    return () => { isActive = false; };
  }, [endpoint]);
}
State Decoupling

Digital Twins & UI State.

Because edge nodes operate on aggressive battery-saving duty cycles, they are often offline and entirely uncontactable for hours at a time. Therefore, the React frontend cannot assume that clicking a "Format NVRAM" button instantly formats the storage.

To solve this synchronization gap, OMEGA models all physical assets on the map as Digital Twins. The React UI manages strictly decoupled state variables for every user action: pending (the operator clicked the button in the UI), synced (the shore gateway queued the command into the DTN mesh), and reported (the hardware edge node physically acknowledged the change and beamed the result back).

Operational Relevance: This tripartite state machine ensures the human operator has absolute, unambiguous clarity on the synchronization gap between the cloud servers and the physical ocean surface.

web/src/types/MissionState.ts
export type SyncStatus = 'pending' | 'synced' | 'reported' | 'failed';

export interface DigitalTwinCommand {
  command_id: string;
  target_node: string;
  intent: string;
  payload: any;
  
  // Tripartite State Tracker
  // pending: UI clicked, waiting on network
  // synced: Shore gateway confirmed receipt
  // reported: Physical edge node confirmed execution over LoRa
  status: SyncStatus;
  
  timestamps: {
    requested_at: string;
    synced_at?: string;
    reported_at?: string;
  };
}
Data Operations

Live Tile Dashboard & Quick Actions.

The portal's Inspect panel abandons the traditional "long-scroll" paradigm in favor of a dense, viewport-fit dashboard. It is engineered for outcome-oriented operations, automatically surfacing the most critical telemetry the second the page loads.

Live Tiles

Real-Time Polling

The Inspect tab continuously polls /v1/measurements/latest every 5 seconds, rendering massive data tiles that immediately reveal the freshest edge node readings, units, and ages. Stale nodes are flagged with a distinct amber border.

Quick Actions

One-Click Outcomes

Six one-click cards (e.g., "My gateway right now" or "Where are my nodes?") automatically pre-fill the Custom Query modals and execute the fetch, completely removing setup friction for common tasks.

Stepped Layout

Modal Drill-Down

Clicking any Live Tile instantly opens a pre-configured Custom Query modal. The workflow is strictly stepped: "1 · Pick a dataset" → "2 · Refine + load" → "3 · View Results / Chart", keeping the interface clean and entirely within the viewport.

3D Rendering

WebGL Bathymetry Rendering.

Plotting millions of acoustic sonar returns in 3D would instantly crash a standard browser. OMEGA utilizes `deck.gl` to push complex oceanographic geometry directly to the GPU.

Layer 1

Hexagon Aggregation

Raw sonar points are aggregated into GPU-rendered hex-bins to visualize dense depth mapping without overlapping geometries.

Layer 2

Volumetric Data Cubes

Interpolated salinity and thermal profiles are rendered as massive 3D volumetric polygons floating beneath the surface.

Layer 3

Live Asset Tracking

Drifting buoys and ROVs are plotted on top of the 3D data cube, calculating their exact physical intersection with the water column.

Real-Time Data

WebSocket Firehose.

While the dashboard relies on resilient HTTP long-polling for poor field connections, it dynamically upgrades to a high-speed WebSocket Firehose when connected to fiber or 5G.

Execution: The Python FastAPI backend maintains a pub/sub event loop. The second a LoRa packet is received, decrypted, and validated, the `websocket_manager` pipes the parsed Binary TLV payload directly into the React DOM at 10Hz, instantly updating the map vectors without a single polling request.

Cartography

Local GeoJSON Uploads & NOAA Charts.

Operators can drag and drop local GeoJSON files directly into the portal to render custom reference zones, exclusion boundaries, or obstacle polygons.

Additionally, the portal includes built-in NOAA chart selectors, instantly replacing the Mapbox satellite basemap with official marine navigation charts to ensure operational compliance during fleet deployment.

Acoustic Operations

The JANUS Test Composer.

Testing acoustic bridges on dry land is notoriously difficult. The Mission Portal features a dedicated JANUS Test Composer, allowing operators to construct NATO-standard acoustic command payloads in the browser, preview the exact binary frame structure, and queue the acoustic transmission directly into the gateway's subsea outbox.

User Experience

Adaptive UX: Desktop vs Field Mode.

A UI designed for a 4K desktop monitor fails on a rain-splattered 6-inch smartphone on a pitching boat. The portal features an Adaptive UX toggle. Standard mode provides dense tabular data arrays; Field mode strips away peripheral data and massively inflates touch targets and high-contrast metric readouts for emergency operations.

Frontend Architecture

React & Vite Mission Portal.

The web/ directory contains the React-based Mission Portal. Built using the modern Vite build tool for instant Hot Module Replacement (HMR) during development, the UI architecture is completely decoupled from the Python gateway backend.

Architecture: It relies entirely on high-speed, asynchronous WebSockets and stateless HTTP polling established with the FastAPI edge router. This architecture allows the UI to stream live edge node telemetry, spatial coordinates, and critical structural alerts directly to the browser DOM at 10Hz, completely bypassing the need for slow, blocking REST API calls.
