WebGL Rendering & Spatial Indexing

Mapbox GL JS Performance.

Rendering thousands of live nodes requires a hardware-accelerated pipeline. The Mission Portal utilizes Mapbox GL JS, pushing the heavy lifting to the client's GPU via WebGL.

To handle extreme density, the system clusters nodes using Supercluster algorithms. Rather than DOM manipulation, points are drawn directly to the WebGL canvas context.

Vector Tile Caching

To save bandwidth on disconnected operations, the frontend aggressively caches Mapbox Vector Tiles (MVT) in the browser's IndexedDB.

Satellite Physics

Global Map Fusion.

To measure Sea Surface Temperature (SST), satellites use radiometers to detect thermal infrared radiation. To model ocean currents, they use radar altimetry—bouncing microwave pulses off the ocean surface to measure minute differences in sea surface height.

The Gateway acts as a caching proxy for these external APIs, normalizing the outputs to prevent exhausting rate limits.

Data Science Workflows

The Visualization Studio.

The Mission Portal is not just a map; it is a full time-series analysis suite. The built-in Visualization Studio generates processed statistical overlays, histograms, anomaly diagnostics, and expanded correlation statistics directly in the browser, allowing oceanographers to perform QA/QC checks without ever opening a Jupyter Notebook.

Cartography

Local GeoJSON Uploads & NOAA Charts.

Operators can drag and drop local GeoJSON files directly into the portal to render custom reference zones, exclusion boundaries, or obstacle polygons.

Additionally, the portal includes built-in NOAA chart selectors, instantly replacing the Mapbox satellite basemap with official marine navigation charts to ensure operational compliance during fleet deployment.

Acoustic Operations

The JANUS Test Composer.

Testing acoustic bridges on dry land is notoriously difficult. The Mission Portal features a dedicated JANUS Test Composer, allowing operators to construct NATO-standard acoustic command payloads in the browser, preview the exact binary frame structure, and queue the acoustic transmission directly into the gateway's subsea outbox.

User Experience

Adaptive UX: Desktop vs Field Mode.

A UI designed for a 4K desktop monitor fails on a rain-splattered 6-inch smartphone on a pitching boat. The portal features an Adaptive UX toggle. Standard mode provides dense tabular data arrays; Field mode strips away peripheral data and massively inflates touch targets and high-contrast metric readouts for emergency operations.

Acoustic Subsea Frames

JANUS Initialization.

This is a simulated NATO-standard JANUS acoustic frame. The payload defines the center_frequency_hz and the rx_level_db. Because underwater acoustics are incredibly low-bandwidth (often ~80 bps), the actual data transferred via the acoustic gateway is usually limited to initialization vectors and critical emergency stop commands.

Bathymetric Execution

Interpolated Depth Soundings.

This is a raw bathymetry sounding envelope. Note the specific metric bathymetry-depth: 12.4m in the SenML records. The gateway consumes these envelopes, storing them as raw points in SQLite. The portal then queries these points and executes Inverse Distance Weighting (IDW) interpolation in real-time, generating a continuous 3D contour map from discrete sonar pings.

Frontend Architecture

React & Vite Mission Portal.

The web/ directory contains the React-based Mission Portal. Built using Vite for instant HMR (Hot Module Replacement) during development, the UI is decoupled entirely from the Python backend. It relies on high-speed WebSockets established with the FastAPI edge router to stream live ASV telemetry and structural alerts directly to the browser DOM at 10Hz, completely bypassing the need for slow REST polling.
