OMEGA Architecture

Data Ingestion & Backend

The software architecture that receives live telemetry from the mesh gateway, normalizes it, and stores it for real-time visualization and bathymetric mapping.

FastAPI Engine

The core ingestion engine is written in Python using FastAPI. It exposes RESTful API endpoints and WebSockets that immediately broadcast incoming data to connected clients with zero latency.

SQLite Timeseries

OMEGA utilizes lightweight SQLite for edge-deployments. The schema is optimized for time-series querying, allowing instant retrieval of fleet tracks, depth profiles, and sensor states.

Data Normalization

The backend performs real-time data cleaning, normalizing timestamps across disparate nodes, applying temperature sound-speed corrections to sonar pings, and generating standard GeoJSON.

Public & Export APIs

At the end of a mission, operators can query the API to export CSVs or KML files containing corrected mapping points. These files are fed into Python scripts for IDW bathymetric surfacing.
