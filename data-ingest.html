OMEGA Architecture

Data Ingestion & Backend

The core software architecture built to ingest live mesh telemetry, perform real-time hydrographic normalization, and expose it via REST and WebSockets.

High-Performance FastAPI Architecture

The central nervous system of the OMEGA gateway is written in modern, asynchronous Python using the FastAPI framework. As the `lora_cbor_bridge` reconstructs binary telemetry from the mesh, it pushes JSON over localhost TCP sockets into the FastAPI backend.

This backend serves three critical functions: Normalization (correcting timestamps and sound-speed profiles), Storage (writing to the SQLite timeseries database), and Broadcasting (pushing updates to the React frontend over WebSockets).

Storage Strategy: SQLite at the Edge

While massive cloud deployments might utilize PostgreSQL or TimescaleDB, OMEGA is designed for disconnected edge deployment. The shoreline gateways run entirely offline on Raspberry Pi or industrial edge PCs, meaning they need a robust, file-based database that won't corrupt upon sudden power loss.

SQLite Timeseries

OMEGA pushes SQLite to its limits by implementing strict Write-Ahead Logging (WAL) and memory-mapped PRAGMA settings. This allows the backend to handle hundreds of concurrent telemetry inserts per second without locking the database, effectively using SQLite as a high-performance timeseries store.

Data Normalization

Before any depth reading is written to the database, the backend passes it through a normalizer. If the Ping sonar reports `14.2 meters` but the onboard temperature sensor detects a strong thermocline, the backend dynamically applies the appropriate sound-speed corrections to derive the true depth.

The Ingestion Code

The FastAPI router strictly enforces data validation using Pydantic models. This ensures that corrupted telemetry or malformed packets from a dying sensor node are rejected before they can pollute the bathymetric database.

FastAPI Routing (Python)

from fastapi import APIRouter, Depends, HTTPException
from pydantic import BaseModel
from typing import Optional

router = APIRouter()

class TelemetryPayload(BaseModel):
    node_id: str
    timestamp: float
    lat: float
    lon: float
    depth_m: Optional[float] = None
    temp_c: Optional[float] = None
    heading: Optional[float] = None

@router.post("/api/v1/telemetry")
async def ingest_telemetry(payload: TelemetryPayload, db = Depends(get_db)):
    # 1. Validate spatial sanity (e.g. not Null Island)
    if payload.lat == 0.0 and payload.lon == 0.0:
        raise HTTPException(status_code=400, detail="Invalid GPS coordinate")
        
    # 2. Normalize and insert into SQLite
    normalized_data = apply_hydrographic_corrections(payload)
    record_id = await db_insert_telemetry(db, normalized_data)
    
    # 3. Broadcast to all active React portal clients instantly
    await websocket_manager.broadcast_json({
        "type": "telemetry_update",
        "data": normalized_data.dict()
    })
    
    return {"status": "success", "record_id": record_id}
