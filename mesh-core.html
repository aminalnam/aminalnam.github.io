Network Graph Theory

Delay Tolerant Networks (DTN).

Standard IP networking relies on the assumption that a direct, unbroken path exists between the source and destination at the time of transmission. In the ocean, this assumption is false.

OMEGA operates a Delay Tolerant Network (DTN) overlay. It models the network as a dynamic graph where edges (connections) appear and disappear. If an Autonomous Surface Vehicle (ASV) captures a packet but cannot reach the shore, it stores the packet in its local NVRAM. The packet is physically carried across the ocean until the ASV drifts into range of a relay buoy, at which point the graph edge reforms and the packet is forwarded.

Routing Algorithms

Route Cost & AutoInterface.

In a mixed-bearer environment, the mesh must decide whether to send a packet over a fast, expensive LTE link, or a slow, free LoRa link. OMEGA utilizes a custom route-scoring system weighing Route Cost (financial/bandwidth penalty) and Route Weight (latency).

For local configuration, it implements a Reticulum-inspired AutoInterface, automatically discovering peers over local IP multicasts without requiring static IP configuration.

Acknowledgment-Aware Outboxes

When a packet is dispatched over the DTN, it sits in an `awaiting_ack` state. If the target node fails to acknowledge within the timeout, the packet falls back to a heavier bearer.

gateway/fleet.py Route Scoring
def calculate_best_route(target_id: str, interfaces: dict) -> str:
    valid_routes = [iface for iface in interfaces.values() if iface.can_reach(target_id)]
    valid_routes.sort(key=lambda x: (x.route_cost, x.route_weight))
    return valid_routes[0].id if valid_routes else None
Binary Optimization

CBOR Payload Compression.

Sending verbose JSON strings over a 900MHz LoRa radio is an egregious waste of airtime. OMEGA compresses API payloads using CBOR (Concise Binary Object Representation).

CBOR strips structural overhead (like quotes, colons, and whitespace) while maintaining the exact key-value mapping. This often reduces payload size by 40-60%, directly translating to reduced radio duty cycles, lower power consumption, and fewer packet collisions over the RF spectrum.

Long-Range Wi-Fi

IEEE 802.11ah HaLow Bridging.

Standard LoRa is restricted to a few hundred bytes per second—useless for streaming video. OMEGA integrates the T-HaLow ESP32-S3 boards. By configuring one node as ap-root and another as sta-client, it creates a transparent TCP/IP network over sub-GHz frequencies (900MHz), allowing the ASV to stream live RTSP camera feeds to the shore gateway from over a kilometer away.

Protocol Bridging

Meshtastic Piggybacking.

OMEGA does not reinvent the wheel. While it has its own routing logic, it includes a robust Python bridge to Meshtastic.

If operators have existing consumer Meshtastic LoRa radios, the OMEGA Gateway can serialize its outbox payloads and push them over the Meshtastic serial API. This allows OMEGA to hijack established consumer meshes to bounce encrypted telemetry packets across vast distances for free.

Remote Operations

Cloud CGNAT Tunneling.

Field operations often rely on marine cellular routers, which are trapped behind massive Carrier-Grade NAT (CGNAT) firewalls. The scripts/tunnel-up.ps1 daemon automates the execution of Cloudflared or Ngrok. It securely punches a reverse TCP tunnel from the ASV gateway out to the public internet, securely exposing the local React Mission Portal to oceanographic teams thousands of miles away.

RF Optimization

LoRa CBOR Frame Compression.

This is a compact_observation payload. As a raw JSON string, it consumes ~448 bytes. Over 900MHz LoRa, this would require a massive Time-On-Air (ToA) penalty and high collision probability. The gateway passes this payload through the Python cbor2 library, crushing the structural overhead (keys, quotes, spaces) down to a ~180 byte binary buffer, allowing it to fit effortlessly inside a single CSS packet window.

Transport Fallback

MQTT Pub/Sub Bridging.

While the DTN is the primary transport, the gateway includes mqtt_bridge.py. If an ASV enters 4G/LTE cellular range, it immediately abandons the slow 1200bps LoRa protocol and begins dumping high-resolution telemetry and camera frames over a high-speed MQTT websocket directly to the cloud.

Low-Level Networking

TCP JSONL Socket Streaming.

For scientific instruments that cannot run an HTTP stack (like legacy Sonde profilers), the gateway exposes raw TCP sockets via tcp_jsonl_bridge.py. Devices can simply pipe newline-delimited JSON strings into port 9000, and the gateway will automatically normalize them into the observation pipeline.

Protocol Engineering

LoRa CBOR Compression.

Sending pure JSON over a 900MHz LoRa link is catastrophic for bandwidth. The lora_cbor_bridge.py engine crushes the SenML Observation Envelopes into Concise Binary Object Representation (CBOR). This shrinks a 400-byte payload into a dense 90-byte binary frame, minimizing the time-on-air and circumventing FCC duty-cycle limitations.
