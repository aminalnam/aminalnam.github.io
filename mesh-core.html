Network Graph Theory

Delay Tolerant Networks (DTN).

Standard IP networking relies on the fundamental assumption that a direct, unbroken, end-to-end path exists between the source and destination at the exact moment of transmission. In the ocean—where nodes are thousands of miles apart, constantly moving, and operating on low-power radios over rolling waves—this assumption is entirely false.

OMEGA operates a Delay Tolerant Network (DTN) overlay to mathematically handle this reality. It models the mesh network as a dynamic, temporal graph where edges (RF or acoustic connections) constantly appear and disappear. If an Autonomous Surface Vehicle (ASV) captures a crucial water-quality measurement but cannot reach the shore gateway, the packet is not simply dropped (as it would be in standard TCP/IP).

Instead, the packet is stored in the ASV's local NVRAM queue. The physical ASV is then treated as a "data mule." It physically carries the packet across the ocean until it drifts into the RF range of a shore station or a relay buoy, at which point the graph edge reforms and the packet is forwarded. This architecture ensures 100% data preservation in extreme latency environments.

Routing Algorithms

Route Cost & AutoInterface.

In a mixed-bearer environment, the mesh gateway must constantly decide whether to send a packet over a fast but incredibly expensive satellite link (Iridium), a fast but short-range cellular link (LTE), or a slow but completely free sub-GHz RF link (LoRa).

OMEGA utilizes a highly customized route-scoring algorithm that dynamically weighs Route Cost (the financial or bandwidth penalty of the transmission) against Route Weight (the physical latency or topological distance of the hop). The Python mesh router ranks every physical interface mapped to a target node.

Operational Use: For local mesh clustering, OMEGA implements a Reticulum-inspired AutoInterface. When a new ASV or Buoy is powered on, it immediately begins broadcasting UDP multicasts over its active hardware interfaces. Nearby nodes intercept these multicasts, extract the cryptographic public key, and instantly map a valid route to the new node without requiring a human to statically assign an IP address. It is true, zero-configuration networking in the field.

Acknowledgment-Aware Outboxes

When a packet is dispatched over the DTN, it sits in an `awaiting_ack` state. If the target node fails to acknowledge within the timeout, the packet falls back to a heavier bearer.

gateway/fleet.py Route Scoring
def calculate_best_route(target_id: str, interfaces: dict) -> str:
    valid_routes = [iface for iface in interfaces.values() if iface.can_reach(target_id)]
    valid_routes.sort(key=lambda x: (x.route_cost, x.route_weight))
    return valid_routes[0].id if valid_routes else None
Binary Optimization

CBOR Payload Compression.

Sending verbose, human-readable JSON strings over a 900MHz LoRa radio is an egregious waste of constrained RF airtime. LoRa provides massive range by utilizing Chirp Spread Spectrum modulation, but at the cost of extremely low bitrates (often <1kbps).

OMEGA compresses all edge payloads using CBOR (Concise Binary Object Representation, RFC 8949). CBOR strips the structural overhead of JSON—such as quotes, colons, brackets, and whitespace—and converts keys and numeric values into strict bit-packed formats, while flawlessly maintaining the exact dictionary mapping.

Relevance: This compression often reduces the byte size of an Observation Envelope by 40-60%. In the field, smaller payloads directly translate to vastly reduced radio duty cycles, lower battery power consumption during transmissions, and a significantly lower mathematical probability of packet collisions over the shared RF spectrum.

Long-Range Wi-Fi

IEEE 802.11ah HaLow Bridging.

Standard 900MHz LoRa is restricted to a few hundred bytes per second—perfect for telemetry, but entirely useless for streaming underwater video feeds. To solve this, OMEGA integrates T-HaLow ESP32-S3 boards.

Execution: The firmware configures one node as an ap-root (Access Point) and another as a sta-client (Station). This establishes a transparent IEEE 802.11ah Wi-Fi connection over sub-GHz frequencies. It creates a massive, long-range TCP/IP tunnel that bypasses standard 2.4GHz Wi-Fi limitations, allowing the ASV to stream live RTSP camera feeds directly to the shore gateway from over a kilometer away.

Protocol Bridging

Meshtastic Piggybacking.

OMEGA does not reinvent the wheel. While it has its own routing logic, it includes a robust Python bridge to Meshtastic.

If operators have existing consumer Meshtastic LoRa radios, the OMEGA Gateway can serialize its outbox payloads and push them over the Meshtastic serial API. This allows OMEGA to hijack established consumer meshes to bounce encrypted telemetry packets across vast distances for free.

Remote Operations

Cloud CGNAT Tunneling.

Field operations often rely on marine cellular routers, which are trapped behind massive Carrier-Grade NAT (CGNAT) firewalls. The scripts/tunnel-up.ps1 daemon automates the execution of Cloudflared or Ngrok. It securely punches a reverse TCP tunnel from the ASV gateway out to the public internet, securely exposing the local React Mission Portal to oceanographic teams thousands of miles away.

RF Optimization

LoRa CBOR Frame Compression.

The compact_observation.json payload, when transmitted as a raw JSON string, consumes approximately 448 bytes. Over a 900MHz LoRa radio operating at SF10, this incurs a massive Time-On-Air (ToA) penalty, legally violating FCC duty cycles and dramatically increasing the probability of a packet collision.

Execution: The gateway intercepts this payload and passes it through the Python cbor2 library. This algorithm aggressively crushes the structural overhead—eliminating all keys, quotes, colons, and spaces—down to a dense ~180-byte binary buffer. This allows the critical telemetry to fit effortlessly inside a single LoRa CSS packet window, ensuring instant transmission and drastically improved range.

Transport Fallback

MQTT Pub/Sub Bridging.

While the Delay Tolerant Network (DTN) LoRa mesh is the primary transport for remote operations, the gateway is highly opportunistic. It includes the mqtt_bridge.py architecture for high-bandwidth offloading.

How it works: If an ASV drifts back into range of a 4G/LTE cellular tower near the coast, it instantly detects the fast connection. It abandons the slow 1200bps LoRa protocol and begins dumping its massive backlog of high-resolution telemetry, acoustic recordings, and camera frames over a high-speed MQTT websocket directly to the cloud, rapidly draining its NVRAM queues before it heads back out to sea.

Low-Level Networking

TCP JSONL Socket Streaming.

Many legacy scientific instruments (like older Sonde profilers or serial-based weather stations) completely lack the hardware to run an HTTP stack or a complex REST client. The gateway accommodates these devices by exposing raw TCP sockets via tcp_jsonl_bridge.py.

Relevance: Legacy devices can simply connect to port 9000 and pipe raw, newline-delimited JSON strings (JSONL) into the open socket. The gateway buffers the stream, detects the line breaks, and automatically normalizes the raw strings into the standard Observation Pipeline, instantly modernizing 10-year-old oceanographic hardware.

Protocol Engineering

LoRa CBOR Compression.

As detailed in the core Mesh sections, sending pure JSON strings over a 900MHz LoRa link is catastrophic for bandwidth. The lora_cbor_bridge.py engine is the workhorse that performs the actual structural crushing.

Execution: It takes the verbose SenML Observation Envelopes and translates them into Concise Binary Object Representation (CBOR). This conversion shrinks a massive 400-byte payload into a dense 90-byte binary frame. This minimizes the physical time-on-air (ToA), legally circumvents FCC transmission duty-cycle limitations, and massively reduces the probability of packet collisions.
